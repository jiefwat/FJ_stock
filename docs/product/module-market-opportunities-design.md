# 股市机会模块设计文档

更新时间：2026-07-10  
适用项目：StockTS / A 股个人投研工作台  
模块目标：发现值得验证的候选股，并解释为什么进入候选、为什么排除、下一步怎么验证。

> 边界：机会模块只做候选发现和验证入口；不直接输出买入，不替代股票分析。

## 1. 参考结论

机会页的有效结构是“市场闸门 -> 策略筛选 -> 风险排除 -> 个股验证 -> 观察跟踪”，不是涨幅榜。

## 2. 分析师标准

机会模块只回答 5 个问题：

1. **今天能不能找机会**：由每日大盘闸门决定。
2. **机会属于哪条策略**：主线强势、资金抱团、放量突破、超跌修复、公告催化、风险排除。
3. **为什么入选**：具体命中条件和证据字段。
4. **为什么可能无效**：追高、缩量、流动性、公告风险、数据缺口、主题伪匹配。
5. **下一步是什么**：进入股票分析、加入观察、忽略、补数据。

策略通道：

| 通道 | 入选条件 | 输出动作 |
| --- | --- | --- |
| 主线强势 | 板块靠前、主题持续、龙头强、成交放大 | 进入验证 |
| 资金抱团 | 成交额/资金流显著，换手健康 | 进入验证或观察 |
| 放量突破 | 突破关键位，量能确认，市场允许 | 进入验证 |
| 超跌修复 | 风险释放，放量企稳，板块回暖 | 只观察或验证 |
| 公告催化 | 财报、订单、回购、政策等正向事件 | 验证事件强度 |
| 风险排除 | 减持、监管、诉讼、质押、退市风险、缺数 | 排除或补数据 |

## 3. 输出格式

候选卡必须短而硬：

```text
股票：000000 示例股份
策略：主线强势 + 放量突破
入选证据：所属主题排名前 5，成交额放大，价格站上关键均线
主要风险：短线涨幅过大，若缩量跌回均线则失效
数据质量：TDX 快照，交易日一致，公告未补齐
下一步：进入股票分析验证六维证据；不直接买入
```

禁止输出：

- 只按涨幅排序。
- 候选没有命中原因。
- 候选没有风险标签。
- 大盘防守时仍鼓励追高。
- 数据缺失时把候选排到前列。

## 4. 页面设计

页面定位：机会雷达和风险过滤器，不是推荐榜。

```text
┌────────────────────────────────────────────┐
│ 机会闸门：允许验证 / 只观察 / 暂停             │
├────────────────────────────────────────────┤
│ 策略通道：主线 / 资金 / 突破 / 修复 / 催化 / 排除 │
├──────────────────────┬─────────────────────┤
│ 候选卡列表            │ 筛选条件 + 数据质量    │
├────────────────────────────────────────────┤
│ 候选对比：证据 / 风险 / 下一步验证             │
└────────────────────────────────────────────┘
```

设计要求：

- 先显示机会闸门，再显示候选。
- 默认按策略通道分组，不混成 Top10。
- 每张候选卡必须有证据、风险、数据质量和下一步。
- 风险排除单独成栏，说明为什么不看。
- 点击候选进入 `股票分析`，必须带入策略和命中证据。
- 市场防守时默认隐藏追高型候选。

## 5. MVP

必须做：

- 读取候选池、市场状态和数据质量。
- 策略通道筛选。
- 候选卡：命中原因、风险、下一步。
- 风险排除列表。
- 一键进入股票分析。

暂不做：

- 黑箱 AI 荐股。
- 无条件买入评级。
- 高频盘中追涨提醒。

## 6. 评分规则

只输出观察优先级，不输出买入评级。

```text
加分：板块强度、成交额、量能、趋势、资金、公告催化
组合加分：与持仓互补，主题不过度重叠，风险预算允许
扣分：追高、缩量、流动性差、公告风险、数据缺失、主题伪匹配
硬闸门：市场暂停、重大风险公告、数据过期、跌停/退市风险
```

标签：

- `可验证`：进入股票分析。
- `只观察`：线索存在，条件不足。
- `风险排除`：不进入候选。
- `待补数据`：不能排序。

## 7. 验收标准

- 每个候选都有策略、证据、风险、数据质量和下一步。
- 机会页不出现无条件买入建议。
- 市场防守或数据过期时自动降级。
- 从候选进入股票分析后，能看到候选来源和命中证据。
