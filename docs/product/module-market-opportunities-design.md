# 股市机会模块界面规格

更新时间：2026-07-13
模块目标：从市场、板块和候选池中筛选值得验证的股票，明确入选原因、排除原因和下一步处理。

## 1. 参考项目与采用原则

参考 GitHub 项目：

- [OpenBB](https://github.com/OpenBB-finance/OpenBB)：研究工作台围绕数据源、筛选、看板组织；StockTS 机会页必须展示候选数据来源和筛选条件。
- [Qlib](https://github.com/microsoft/qlib)：量化研究包含 alpha 寻找、风险建模、组合优化；StockTS 机会页先找候选，再进入风险验证。
- [FinRL](https://github.com/AI4Finance-Foundation/FinRL)：流程包含数据处理、技术指标、测试/交易分层；StockTS 机会页只做候选，不做执行。
- [FinRL-X](https://github.com/AI4Finance-Foundation/FinRL-Trading)：策略输出需要风险覆盖层；StockTS 机会页必须有风险排除区。

采用原则：机会页只输出观察优先级，不输出买入建议。

## 2. 页面内容

| 区域 | 必须展示 | 数据来源/口径 | 展示形式 |
| --- | --- | --- | --- |
| 机会状态 | 市场状态、机会状态、可用策略、数据质量、扫描范围 | 市场快照、候选池、数据质量 | 首屏状态卡 |
| 策略通道 | 主线强势、资金抱团、放量突破、超跌修复、公告催化、风险排除 | 候选理由和市场状态 | 当前展示分组，后续可做 Tabs |
| 候选列表 | 代码、名称、行业/概念、策略、入选证据、风险、数据日期、下一步 | `CandidatePoolReport` | 卡片或主表格 |
| 筛选条件 | 当前策略条件、阈值、命中字段、排序口径 | 候选生成逻辑 | 右侧面板或折叠区 |
| 风险排除 | 股票、排除原因、风险类型、数据状态 | 风险规则、公告、数据质量 | 单独列表 |
| 板块方向 | 行业/概念强度、扩散、代表股 | `SectorAnalysisReport` | 表格 |
| 情绪温度 | 涨停、跌停、赚钱效应、亏钱效应 | 市场情绪数据 | 指标卡 |
| 数据质量 | 扫描数量、预筛数量、补强数量、Provider、缺失字段 | 快照元数据 | 状态条 |

## 3. 桌面布局

```text
┌──────────────────────────────────────────────────────┐
│ 机会状态：市场状态 / 可用策略 / 数据质量 / 扫描范围    │
├──────────────────────────────────────────────────────┤
│ 策略通道：主线 / 资金 / 突破 / 修复 / 催化 / 排除       │
├──────────────────────────┬───────────────────────────┤
│ 候选列表                  │ 筛选条件 / 数据质量         │
├──────────────────────────┬───────────────────────────┤
│ 板块方向                  │ 情绪温度                   │
├──────────────────────────────────────────────────────┤
│ 风险排除：排除原因 / 风险类型 / 数据状态               │
└──────────────────────────────────────────────────────┘
```

布局要求：

- 机会状态在首屏顶部，先判断是否允许找机会。
- 候选列表是主体，但每行/卡必须带风险和下一步。
- 筛选条件和数据质量靠近候选列表，不放到底部。
- 风险排除单独成区，不混在候选列表里。

## 4. 移动布局

顺序固定：

1. 机会状态。
2. 策略通道。
3. 候选卡片。
4. 筛选条件。
5. 风险排除。
6. 板块方向。
7. 情绪温度。
8. 数据质量。

移动端要求：

- 候选卡片字段固定：股票、策略、入选证据、风险、数据日期、下一步。
- 策略通道横向滚动，不堆成多行。
- 风险排除默认展示前 5 条。

## 5. 排版规则

| 元素 | 规则 |
| --- | --- |
| 标题 | 使用“机会状态、策略通道、候选列表、筛选条件、风险排除” |
| 候选卡 | 每卡最多 6 个字段；入选证据和风险必须同时出现 |
| 表格 | 候选表列顺序固定：股票、策略、证据、风险、数据日期、下一步 |
| 标签 | 可验证、只观察、风险排除、待补数据 |
| 颜色 | 可验证=中性强调，观察=黄，排除=灰/红，待补数据=灰 |
| 文案 | 不出现买入评级；按钮使用“分析”“加入观察”“忽略”“补数据” |

## 6. 操作和状态

| 操作/状态 | 页面处理 |
| --- | --- |
| 选择策略通道 | 当前展示对应分组；后续实现筛选后更新候选列表和筛选条件 |
| 点击“分析” | 进入股票分析，带入策略和入选证据 |
| 加入观察 | 写入观察清单；未实现时不展示该按钮 |
| 忽略候选 | 从当前视图隐藏；未实现时不展示该按钮 |
| 待补数据 | 不参与排序，只提示缺失字段 |
| 市场防守 | 默认隐藏或降级追高类候选 |
| 数据过期 | 全部候选降级为待补数据或只观察 |

## 7. 与当前实现对齐

| 项目 | 当前实现 | 建议 | 以哪个为准 |
| --- | --- | --- | --- |
| 工作区 | `opportunity`，显示“热点机会” | 产品口径可改为“股市机会” | 当前代码和测试为准 |
| 策略通道 | 当前是展示型分组 | 后续可做真正 Tabs/筛选 | 当前代码为准 |
| 候选操作 | 当前主要是进入个股分析 | 加入观察、忽略、补数据为后续项 | 当前代码为准 |
| 风险排除 | 当前展示原因 | 后续做独立排除列表 | 当前代码为准 |
| 命名统一 | 文档使用“股市机会” | 若改 UI 名称，需同步导航和测试 | 改造时以本规格为目标 |

## 8. 验收标准

- 每个候选都有策略、入选证据、风险、数据日期和下一步。
- 筛选条件可见，不是黑箱列表。
- 风险排除单独展示，不隐藏原因。
- 点击候选进入股票分析后，来源上下文完整保留。
- 市场防守或数据过期时，不显示追高型候选为高优先级。

## 9. 2026-07-13 已实现升级

- 首屏新增唯一“机会总闸门”，直接继承大盘风险预算并展示扫描范围、证据就绪数和可验证数量。
- 候选筛选统一为五阶段证据漏斗：数据校验、市场门槛、风险排除、证据补强、个股验证。
- 候选只使用“可验证、只观察、风险排除、待补数据”，不再输出“推荐买入”或“推荐股票”。
- 每个候选同时展示支持证据、最大反证、数据日期、排除原因和下一步验证。
- 市场与数据风险进入独立风险登记表；板块、资金、新闻和公告只作为证据，不直接改写买卖结论。
- 行情 stale/blocked 时，可验证数量归零，全部候选转为待补数据并停止排序。
