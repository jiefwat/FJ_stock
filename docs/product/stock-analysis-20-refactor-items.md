# 股票分析工作台 20 项改造清单

> 来源：5 位资深工程师视角（软件交互师、股票分析师、页面设计师、软件开发、软件测试）。本清单用于把讨论结果转成可追踪的工程事项。

## 软件交互

1. 模块单一目标：每个模块只回答一个核心问题。状态：已落地。
2. 入口按工作流排序：先市场，再选股/持仓，再个股，再复盘。状态：已落地。
3. 减少重复头部：避免每个模块反复出现相同总控区。状态：已落地。
4. 公开只读边界：线上页面隐藏写操作，避免误改持仓。状态：已落地。

## 股票分析

5. 风险先行：先判断能不能行动，再看机会。状态：已落地。
6. 证据链闭环：趋势、风险、公告、操作条件合并展示。状态：已落地。
7. 操作条件化：只输出条件触发，不输出确定性承诺。状态：已落地。
8. 复盘计划化：日报沉淀明日计划、风险复核、数据复核。状态：进行中。

## 页面设计

9. 简洁卡片密度：只保留关键数据和动作。状态：进行中。
10. 去旁白文案：删除解释性套话。状态：进行中。
11. 关键数字优先：热度、仓位、风险、日期优先可见。状态：已落地。
12. 移动端可读：继续优化小屏表格和模块层级。状态：待做。

## 软件开发

13. 数据可信度：样例、降级、暂停三态统一。状态：已落地。
14. TDX 主源收口：公开 Web 固定 TDX MCP 快照。状态：已落地。
15. Tushare 兜底：Provider 已接入，待配置 Token 和线上依赖。状态：进行中。
16. 跨市场接口预留：港股/美股保留数据源路由。状态：待做。

## 软件测试

17. 自动化回归：Web 决策、风险闸门、数据暂停均有测试。状态：已落地。
18. 只读安全测试：线上写操作必须 403。状态：已落地。
19. 数据暂停测试：候选价格不可靠时隐藏假排名。状态：已落地。
20. 模块点击验收：补模块级内容可见性和路由回归测试。状态：已落地。
