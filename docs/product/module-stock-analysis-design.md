# 股票分析模块界面规格

更新时间：2026-07-11
模块目标：对单只股票进行多维证据分析，输出当前状态、主要证据、主要风险、触发条件、失效条件和组合影响。

## 1. 参考项目与采用原则

参考 GitHub 项目：

- [TradingAgents](https://github.com/tauricresearch/tradingagents)：用基本面、情绪、新闻、技术、交易员、风控等角色拆解决策；StockTS 保留结构化证据链和风险角色，但界面不展示冗长辩论。
- [OpenBB](https://github.com/OpenBB-finance/OpenBB)：研究看板要能复用统一数据源；StockTS 个股页必须展示 Provider、交易日和缺失字段。
- [Qlib](https://github.com/microsoft/qlib)：研究链路包含数据处理、风险建模、组合优化；StockTS 单股结论必须接到组合影响。
- [FinRL](https://github.com/AI4Finance-Foundation/FinRL)：技术指标、数据预处理和验证流程分离；StockTS 不让单一指标决定结论。

采用原则：个股页是证据验证界面，不是长研报页，不用评分替代证据。

## 2. 页面内容

| 区域 | 必须展示 | 数据来源/口径 | 展示形式 |
| --- | --- | --- | --- |
| 股票输入 | 代码/名称、解析结果、交易所/市场 | `symbols.py`、请求参数 | 顶部搜索条 |
| 股票摘要 | 代码、名称、现价、涨跌幅、成交额、交易日、Provider | 个股行情 | 首屏指标卡 |
| 当前结论 | 处理状态、置信度、主要证据、主要风险、触发条件、失效条件、禁止动作 | `TradePlan`、深度分析 | 首屏结论卡 |
| 技术面 | 趋势、均线、支撑压力、量价、波动、RSI/MACD | K 线和技术指标 | 证据卡或表格行 |
| 基本面 | 营收、利润、现金流、资产负债、ROE、估值 | 基本面/估值快照 | 证据卡；缺失时标记 |
| 资金面 | 成交额、换手、量能比、资金流、龙虎榜可用项 | 资金和盘口数据 | 证据卡 |
| 消息公告 | 催化、风险公告、财报、监管、减持、诉讼、质押 | 新闻/公告 | 风险优先列表 |
| 板块主题 | 行业、概念、板块强度、同主题对比 | 板块报告 | 证据卡 |
| 成本位置 | 是否持仓、成本、浮盈亏、仓位、止损距离 | 持仓账本 | 单独卡片 |
| 来源上下文 | 来自持仓或机会、策略、入选证据 | URL 参数、持仓入口 | 小型信息条 |
| 数据质量 | 数据源、交易日、缺失字段、降级原因 | `DataQualityView` | 固定状态区 |

## 3. 桌面布局

```text
┌──────────────────────────────────────────────────────┐
│ 股票输入：代码/名称 / 解析结果 / 数据源               │
├──────────────────────────────────────────────────────┤
│ 股票摘要：现价 / 涨跌幅 / 成交额 / 交易日              │
├──────────────────────────────────────────────────────┤
│ 当前结论：状态 / 主要证据 / 主要风险 / 触发 / 失效      │
├──────────────────────┬───────────────────────────────┤
│ 六类证据              │ 成本位置 / 组合影响             │
│ 技术/基本/资金/消息/板块│                               │
├──────────────────────┬───────────────────────────────┤
│ 风险与反证            │ 数据质量 / 原始证据              │
└──────────────────────┴───────────────────────────────┘
```

布局要求：

- 当前结论必须在首屏，不放在证据区之后。
- 成本位置独立显示；未持仓时显示“未持仓”。
- 证据区最多 6 张卡或 6 行表格，每项只展示结论、依据、状态。
- 原始新闻、公告、财务字段放折叠区。
- 从机会进入时，来源上下文放在当前结论下方。

## 4. 移动布局

顺序固定：

1. 股票输入。
2. 股票摘要。
3. 当前结论。
4. 成本位置。
5. 六类证据。
6. 风险与反证。
7. 数据质量。
8. 原始证据。

移动端要求：

- 当前结论不折叠。
- 六类证据每类一张卡，默认只显示 2-3 行。
- 原始证据默认折叠。

## 5. 排版规则

| 元素 | 规则 |
| --- | --- |
| 标题 | 使用“当前结论、技术面、基本面、资金面、消息公告、板块主题、成本位置” |
| 状态标签 | 可观察、等待触发、降低风险、暂停、数据不足 |
| 数字 | 价格、成交额、成本、仓位使用等宽数字 |
| 证据 | 每类证据固定三列：状态、依据、风险/缺口 |
| 风险 | 风险公告、数据缺口、破位条件优先展示 |
| 评分 | 可以展示，但不能单独作为结论；必须配合证据和风险 |

## 6. 操作和状态

| 操作/状态 | 页面处理 |
| --- | --- |
| 搜索股票 | 刷新摘要、结论、证据和数据质量 |
| 从持仓进入 | 带入成本、仓位、浮盈亏和止损距离 |
| 从机会进入 | 带入策略、入选证据和候选来源 |
| 点击证据 | 展开原始字段或来源 |
| 数据缺失 | 降低置信度，并标记缺失维度 |
| 重大风险公告 | 状态至少降级为等待触发或暂停 |

## 7. 与当前实现对齐

| 项目 | 当前实现 | 建议 | 以哪个为准 |
| --- | --- | --- | --- |
| 工作区 | `stock`，显示“个股分析” | 保持 | 代码为准 |
| 决策链 | 架构要求保留 TradingAgents/DSA 风格链路 | 可折叠，但不能删除 | 架构文档和当前实现为准 |
| 证据维度 | 当前已有 6 维判断 | 保持 | 架构文档为准 |
| 评分字段 | 当前显示综合机会评分 | 降低视觉权重，不能替代证据 | 当前代码为准，文档为改造目标 |
| 加入观察/添加持仓 | 当前不一定完整 | 作为后续操作建议 | 当前代码为准 |

## 8. 验收标准

- 每只股票都有当前结论、主要证据、主要风险、触发条件和失效条件。
- 六类证据都有状态；缺失项必须可见。
- 从持仓或机会进入时，来源上下文完整保留。
- 评分不能脱离证据独立展示为结论。

## 9. 2026-07-13 已实现升级

- 首屏由“机会评分”切换为研究状态、置信度、核心矛盾、最强证据和最强反证。
- 新增投资备忘录，明确区分公司/生意、经营质量、估值与预期、预期差。
- 新增乐观、基准、悲观三情景，只表达前提、信号、动作和证伪，不承诺收益或强行给目标价。
- 六类证据按来源、日期和 complete/degraded/missing 状态审计；TradingAgents 角色链保留为支持层。
- 只有绝对 PE/PB 而没有历史分位或行业对比时，只描述绝对估值水平，不输出“低估”。
- 单期财务不得声称趋势改善；公告标题规则必须注明未代替原文复核；未持仓不得生成成本优势或处置结论。
