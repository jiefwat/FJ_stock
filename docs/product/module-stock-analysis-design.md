# 股票分析模块设计文档

更新时间：2026-07-10  
适用项目：StockTS / A 股个人投研工作台  
模块目标：对单只股票给出证据链判断，明确可观察、等待触发、降低风险或暂停。

> 边界：只输出条件化研究结论；不得把分数包装成确定性买卖建议。

## 1. 参考结论

专业股票分析至少要覆盖基本面、技术面、估值、资金/成交、新闻公告、行业环境和风险。单一指标不能作为结论。

## 2. 分析师标准

股票分析只回答 5 个问题：

1. **结论是什么**：可观察、等待触发、降低风险、暂停。
2. **最强证据是什么**：技术、基本面、资金、消息、板块、成本中的哪一项最有力。
3. **最大反证是什么**：趋势破位、估值过高、资金流出、风险公告、主题退潮、成本压力。
4. **触发和失效是什么**：价格、量能、均线、公告、市场状态。
5. **对当前组合有什么影响**：是否已持仓、仓位上限、是否增加集中度。

六维证据必须同时给状态：支持、反对、缺口。

| 维度 | 看什么 | 不允许 |
| --- | --- | --- |
| 技术面 | 趋势、均线、支撑压力、量价、波动 | 只看涨跌幅下结论 |
| 基本面 | 营收、利润、现金流、资产负债、ROE、估值 | 缺财务数据仍强判 |
| 资金面 | 成交额、换手、量能比、资金流、龙虎榜可用项 | 把单日资金当长期逻辑 |
| 消息公告 | 催化、风险、财报、监管、减持、诉讼、质押 | 忽略重大风险公告 |
| 概念板块 | 行业、主题、板块强度、同主题对比 | 用概念名替代业绩证据 |
| 成本位置 | 持仓成本、浮盈亏、止损距离、仓位影响 | 未持仓却显示成本判断 |

## 3. 输出格式

首屏必须输出“单股判决卡”：

```text
最终动作：等待触发
最强证据：放量站上关键均线，所属主题仍在市场前排
最大反证：估值偏高，公告数据未补齐
触发条件：回踩不破支撑且成交额维持放大
失效条件：跌破支撑、主题退潮、出现减持/监管风险公告
组合影响：已持仓，仓位不超过组合 12%，不主动摊薄
```

禁止输出：

- 不同股票套同一句结论。
- 只有评分，没有证据和反证。
- 没有失效线的行动建议。
- 数据缺口被当作中性，而不是降低置信度。

## 4. 页面设计

页面定位：证据审判席，不是长研报。

```text
┌────────────────────────────────────────────┐
│ 股票输入：代码/名称 + 当前持仓状态             │
├────────────────────────────────────────────┤
│ 单股判决卡：动作 / 最强证据 / 最大反证 / 失效线 │
├──────────────┬──────────────┬──────────────┤
│ 交易快照      │ 成本位置      │ 数据质量      │
├────────────────────────────────────────────┤
│ 六维证据墙：技术 / 基本面 / 资金 / 消息 / 板块 / 成本 │
├──────────────────────┬─────────────────────┤
│ 多空反证              │ 执行边界             │
└──────────────────────┴─────────────────────┘
```

设计要求：

- 判决卡放首屏，不让用户先读指标海。
- 六维证据每项最多 3 条，必须标明支持/反对/缺口。
- 从持仓进入时显示成本、浮盈亏和仓位影响。
- 从机会进入时显示候选来源和命中策略。
- 重大风险公告出现时，动作至少降级为等待或暂停。
- 原始新闻、公告、财务字段放抽屉，不挤首屏。

## 5. MVP

必须做：

- 股票搜索/名称解析。
- 单股判决卡、交易快照、六维证据、数据质量。
- 持仓成本接入。
- 触发线、失效线、禁止动作。

暂不做：

- 自动交易。
- 长篇 AI 研报作为主结论。
- 没有数据支撑的主观评级。

## 6. 验收标准

- 每只股票的结论必须由具体证据驱动。
- 每个动作都有触发条件、失效条件和组合影响。
- 未持仓显示“未持仓”，不展示成本判断。
- 数据缺失必须降低置信度并可见。
