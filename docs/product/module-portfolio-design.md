# 我的持仓模块设计文档

更新时间：2026-07-10  
适用项目：StockTS / A 股个人投研工作台  
模块目标：把持仓从股票列表变成风险处置队列，明确今天先处理什么。

> 边界：只做持仓研究和风险预算，不连接真实券商，不在公开只读环境暴露写操作。

## 1. 参考结论

成熟 Portfolio 页面不只展示盈亏，还要展示成本、仓位、行业暴露、集中度、风险、提醒和组合透视。

参考：

- [Yahoo Finance Portfolios](https://finance.yahoo.com/portfolios/)
- [Yahoo Finance Watchlists](https://finance.yahoo.com/watchlists/)
- [Morningstar Portfolio X-Ray](https://www.morningstar.com/investment-management/portfolio-x-ray)
- [FINRA: Asset Allocation and Diversification](https://www.finra.org/investors/investing/investing-basics/asset-allocation-diversification)
- [Investor.gov: Diversification](https://www.investor.gov/introduction-investing/investing-basics/glossary/diversification)

## 2. 分析师标准

持仓页只回答 4 个问题：

1. **组合是否健康**：总盈亏、现金比例、集中度、风险等级。
2. **风险在哪里**：单股过重、行业过重、跌破成本、数据缺失。
3. **今天先处理什么**：止损、减仓、观察、补数据、继续持有。
4. **每只股票的边界是什么**：成本、触发线、失效线、仓位上限。

必须展示：

| 信息 | 必要字段 | 用途 |
| --- | --- | --- |
| 组合总览 | 总市值、总成本、浮盈亏、当日盈亏、现金比例 | 看组合状态 |
| 风险预算 | 单股上限、行业上限、目标现金、风险等级 | 控制暴露 |
| 处置队列 | 必须处理、观察、可继续、待补数据 | 今天先做什么 |
| 持仓明细 | 代码、名称、数量、成本、现价、盈亏、仓位、行业 | 账本依据 |
| 执行边界 | 止损线、减仓线、加仓触发、禁止动作 | 条件化行动 |
| 数据质量 | 持仓账本、行情日期、缺失报价、Provider | 防止错判 |

## 3. 输出格式

首屏必须输出“组合处置摘要”：

```text
组合状态：防守型，目标现金 40%-55%
最大风险：机器人主题暴露过高，2 只持仓弱于成本
今日先处理：A 核对止损线，B 禁止摊薄，C 等待放量确认
可继续持有：D 趋势未破，但不加仓
数据限制：E 行情日期落后，建议降级为补数据后复核
```

禁止输出：

- 只列持仓表，不给处置优先级。
- 亏损股默认建议补仓。
- 只看单股，不看组合集中度。
- 数据缺失时仍给加仓/减仓动作。

## 4. 页面设计

页面定位：组合风控台，不是 Excel。

```text
┌────────────────────────────────────────────┐
│ 组合健康灯：状态 / 目标现金 / 最大风险         │
├───────────┬───────────┬───────────┬────────┤
│ 总市值     │ 总盈亏     │ 现金比例   │ 集中度  │
├──────────────────────┬─────────────────────┤
│ 处置队列              │ 行业/主题暴露         │
├────────────────────────────────────────────┤
│ 持仓明细：动作 / 成本 / 触发线 / 风险标签       │
├────────────────────────────────────────────┤
│ 账本操作：添加 / 编辑 / 删除 / 导入             │
└────────────────────────────────────────────┘
```

设计要求：

- 默认按风险优先级排序，不按股票代码排序。
- 首屏先显示处置队列，再显示明细表。
- 每只持仓必须有动作标签和下一次复核条件。
- 行业/主题暴露超过阈值必须高亮。
- 公开只读环境隐藏或禁用账本写操作。
- 移动端把明细表改成持仓卡片。

## 5. MVP

必须做：

- 读取 `holdings.csv` 或账号隔离账本。
- 组合总览、风险预算、处置队列、持仓明细。
- 添加、编辑、删除持仓；公开环境只读禁用。
- 点击持仓进入 `股票分析` 并带入成本。

暂不做：

- 真实券商同步。
- 自动下单。
- 复杂收益归因。

## 6. 验收标准

- 3 秒内知道今天先处理哪只持仓。
- 每只持仓都有成本位置、风险标签、动作、失效线。
- 组合集中度风险必须可见。
- 写操作不会在公开只读环境生效。
