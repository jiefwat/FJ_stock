# 我的持仓模块界面规格

更新时间：2026-07-11
模块目标：展示组合风险、持仓处理优先级、单股成本位置和操作边界。

## 1. 参考项目与采用原则

参考 GitHub 项目：

- [OpenBB](https://github.com/OpenBB-finance/OpenBB)：研究数据可被工作台、Excel、API 等多端消费；StockTS 持仓页保持账本来源和数据日期可见。
- [Qlib](https://github.com/microsoft/qlib)：覆盖风险建模和组合优化；StockTS 持仓页优先展示风险预算和集中度。
- [FinRL](https://github.com/AI4Finance-Foundation/FinRL)：包含 portfolio allocation 等场景；StockTS 采用“仓位约束先于个股动作”。
- [FinRL-X](https://github.com/AI4Finance-Foundation/FinRL-Trading)：用组合权重和风险覆盖层连接策略与执行；StockTS 采用“目标仓位/低风险仓位”表达，不展示不存在的真实现金。

采用原则：持仓页是组合风控界面，不是交易下单界面。

## 2. 页面内容

| 区域 | 必须展示 | 数据来源/口径 | 展示形式 |
| --- | --- | --- | --- |
| 组合摘要 | 总市值、总成本、浮盈亏、当日盈亏、健康度、目标现金/低风险仓位 | `PortfolioAnalysisReport`、`PortfolioAdvice` | 首屏主卡片 |
| 风险预算 | 单股集中度、行业集中度、目标现金/低风险仓位、超限项 | 持仓权重、行业权重 | 条形指标 |
| 处理队列 | 必须处理、重点观察、可继续持有、待补数据 | `PositionAdvice` 和风险规则 | 四列队列 |
| 持仓明细 | 代码、名称、数量、成本、现价、盈亏、仓位、行业、状态、数据日期 | 持仓账本 + 行情 | 主表格 |
| 行业暴露 | 行业/概念、持仓数量、仓位占比、集中度状态 | 持仓行业字段 | 横向条形图或紧凑表 |
| 操作边界 | 止损线、减仓线、加仓触发、仓位上限、禁止动作 | `TradePlan`、组合建议 | 边界表 |
| 账本状态 | 账本路径、账号隔离、公开只读、可编辑状态 | Web auth/readonly 状态 | 状态条 |
| 数据质量 | Provider、行情日期、缺失报价 | 数据质量层 | 页尾或侧栏 |

## 3. 桌面布局

```text
┌──────────────────────────────────────────────────────┐
│ 组合摘要：总市值 / 总盈亏 / 健康度 / 目标现金/低风险   │
├─────────────────────────┬────────────────────────────┤
│ 风险预算                 │ 账本状态                    │
├──────────────────────────────────────────────────────┤
│ 处理队列：必须处理 / 重点观察 / 可继续持有 / 待补数据   │
├─────────────────────────┬────────────────────────────┤
│ 持仓明细表               │ 行业暴露                    │
├──────────────────────────────────────────────────────┤
│ 操作边界 / 数据质量                                      │
└──────────────────────────────────────────────────────┘
```

布局要求：

- 组合摘要和风险预算在首屏。
- 处理队列必须在持仓明细表之前。
- 持仓明细默认按处理优先级排序，不按股票代码排序。
- 账本操作区只在私有/本地模式展示，公开只读模式只显示状态。

## 4. 移动布局

顺序固定：

1. 组合摘要。
2. 风险预算。
3. 处理队列。
4. 持仓卡片。
5. 行业暴露。
6. 操作边界。
7. 账本状态和数据质量。

移动端要求：

- 持仓表转卡片，每张卡显示：名称、代码、盈亏、仓位、成本位置、处理状态。
- 处理队列不折叠。
- 编辑入口放在卡片底部，仅私有/本地模式显示。

## 5. 排版规则

| 元素 | 规则 |
| --- | --- |
| 标题 | 使用“组合摘要、风险预算、处理队列、持仓明细、行业暴露” |
| 数字 | 盈亏、仓位、成本、现价使用等宽数字；盈亏颜色遵守 A 股红绿语义 |
| 队列 | 四列卡片，卡片字段固定：股票、状态、原因、复核条件 |
| 表格 | 明细表列数控制在 8-10 列；次要字段放展开区 |
| 风险 | 超限、亏损扩大、缺报价必须高亮 |
| 文案 | 不使用鼓励性或解释性长句；只保留状态和处理条件 |

## 6. 操作和状态

| 操作/状态 | 页面处理 |
| --- | --- |
| 点击持仓 | 进入股票分析，带入代码、成本、仓位 |
| 新增/编辑/删除 | 仅私有/本地模式可用；公开只读模式隐藏或禁用 |
| 导入交易流水 | 更新移动加权成本；必须显示导入结果 |
| 待补数据 | 不输出加仓/减仓建议，只显示缺失字段 |
| 单股超限 | 进入“必须处理”或“重点观察” |
| 行业超限 | 行业暴露区高亮，并影响新机会判断 |

## 7. 与当前实现对齐

| 项目 | 当前实现 | 建议 | 以哪个为准 |
| --- | --- | --- | --- |
| 工作区 | `portfolio`，显示“我的持仓” | 保持 | 代码为准 |
| 现金字段 | 当前无真实现金余额，只有目标现金/低风险仓位 | 不展示实际现金比例 | 当前数据模型为准 |
| 写操作 | 私有模式可编辑，公开模式只读 | 保持 | 安全规则和测试为准 |
| 行业暴露跳转 | 当前以展示为主 | 后续可点击进入机会模块 | 当前代码为准，跳转为改造项 |
| 导出功能 | 文档建议导出，当前不一定完整 | 不写成已实现能力 | 当前代码为准 |

## 8. 验收标准

- 首屏能看出组合风险等级、目标现金/低风险仓位和优先处理持仓。
- 每只持仓都有状态、原因、成本位置、数据日期和复核条件。
- 公开只读模式下没有可执行写操作。
- 数据缺失时不输出加仓或减仓建议。
