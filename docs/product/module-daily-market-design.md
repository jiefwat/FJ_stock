# 每日大盘模块界面规格

更新时间：2026-07-11  
模块目标：判断当日市场状态、风险等级、可关注方向和数据可信度，作为进入持仓、个股和机会模块的前置闸门。

## 1. 参考项目与采用原则

参考 GitHub 项目：

- [OpenBB](https://github.com/OpenBB-finance/OpenBB)：数据接入与研究看板分离，同一数据可供不同终端消费；StockTS 采用“数据状态必须显式展示”。
- [Qlib](https://github.com/microsoft/qlib)：量化研究链路覆盖数据处理、回测、风险建模、组合优化；StockTS 采用“市场状态先约束风险，再进入个股判断”。
- [FinRL](https://github.com/AI4Finance-Foundation/FinRL)：强调数据预处理、技术指标、训练/测试/交易流程；StockTS 采用“先数据质量，再策略验证”。
- [FinRL-X](https://github.com/AI4Finance-Foundation/FinRL-Trading)：用风险覆盖层约束策略输出；StockTS 采用“市场风险优先级高于机会提示”。

采用原则：页面只做市场判断，不做资讯聚合，不直接给单股操作。

## 2. 页面内容

| 区域 | 必须展示 | 数据来源/口径 | 展示形式 |
| --- | --- | --- | --- |
| 市场摘要 | 市场状态、风险等级、目标现金/风险暴露、交易日、更新时间 | `MarketSnapshot`、数据质量状态 | 首屏主卡片 |
| 指数表现 | 主要指数、涨跌幅、成交额、关键位置 | 指数行情 | 3-6 张紧凑指标卡或横向表格 |
| 市场宽度 | 上涨/下跌/平盘家数、涨停、跌停、炸板、成交额变化 | 大盘宽度和情绪数据 | 三列指标区 |
| 板块方向 | 行业/概念、强度、持续性、资金、代表股 | `SectorAnalysisReport` | 排名表，不做买入结论 |
| 风险项 | 缩量、破位、跌停扩散、主线退潮、数据过期、重大事件 | 大盘风险、数据质量、公告/日历 | 风险列表，优先显示 |
| 市场异动 | 涨幅、跌幅、放量、换手、资金流入 | 候选池/行情快照 | 二级表格，可折叠 |
| 数据质量 | Provider、交易日、刷新时间、缺失字段、样例/降级/过期 | `DataQualityView`、pipeline 状态 | 固定状态条或页尾区 |

## 3. 桌面布局

```text
┌──────────────────────────────────────────────────────┐
│ 市场摘要：状态 / 风险等级 / 目标现金 / 交易日 / 数据源 │
├─────────────────┬─────────────────┬──────────────────┤
│ 指数表现         │ 市场宽度         │ 风险项            │
├──────────────────────────────────────────────────────┤
│ 板块方向：行业/概念 / 强度 / 持续性 / 资金 / 代表股     │
├──────────────────────────┬───────────────────────────┤
│ 市场异动                 │ 数据质量与缺失字段          │
└──────────────────────────┴───────────────────────────┘
```

布局要求：

- 首屏高度内必须看到市场摘要、风险项、至少 3 个核心指标。
- 风险项放在首屏右侧或第二列，不放到底部。
- 板块方向位于指标区之后，避免用户跳过市场风险直接看机会。
- 数据质量必须常驻，不只放在详情页。

## 4. 移动布局

顺序固定：

1. 市场摘要。
2. 风险项。
3. 市场宽度。
4. 指数表现。
5. 板块方向。
6. 市场异动。
7. 数据质量。

移动端要求：

- 表格转为卡片，每张卡最多 4 个字段。
- 风险项不折叠。
- 板块方向默认展示 Top 5，其余折叠。

## 5. 排版规则

| 元素 | 规则 |
| --- | --- |
| 标题 | 使用业务名，不使用比喻名；例如“市场摘要”“风险项”“板块方向” |
| 主数字 | 只突出市场状态、风险等级、热度/宽度、交易日；不同时放大过多数字 |
| 表格 | 行高紧凑，列名固定；涨跌幅、成交额、持续性右对齐或等宽数字 |
| 颜色 | 红=强/上涨，绿=弱/下跌，黄=观察，灰=缺数/降级 |
| 文案 | 每个区块只保留字段名、状态和结论，不写解释性长句 |
| 密度 | 首屏 6-9 个关键字段；详情放折叠区 |

## 6. 操作和状态

| 操作/状态 | 页面处理 |
| --- | --- |
| 点击板块方向 | 进入机会模块，并带入行业/概念筛选条件 |
| 点击风险项 | 展开影响范围、触发条件和相关数据 |
| 点击指数 | 展开该指数近期走势和关键位置 |
| 数据过期 | 市场状态降级，不显示进攻类结论 |
| 数据不足 | 所有方向标记为待验证 |
| 市场防守 | 机会入口降级，持仓风险入口前置 |

## 7. 与当前实现对齐

| 项目 | 当前实现 | 建议 | 以哪个为准 |
| --- | --- | --- | --- |
| 工作区 | `market`，显示“每日大盘” | 保持 | 代码为准 |
| 页面标题 | “每日大盘 · 仓位闸门” | 后续可改为“每日大盘”+“市场摘要” | 短期代码为准，改名需同步测试 |
| 数据质量 | 嵌入大盘/全局状态，不是顶级工作区 | 保持嵌入 | 代码和架构文档为准 |
| 机会跳转 | 当前跳到“热点机会” | 后续如统一为“股市机会”，需同步导航和测试 | 当前代码为准 |
| 展示命名 | 代码仍有偏展示化名称 | 后续替换为中性字段名 | 本文档作为改造目标 |

## 8. 验收标准

- 首屏 3 秒内能判断市场状态、风险等级、是否适合继续看机会。
- 每个板块方向都有行业/概念名称、强度、持续性、数据源。
- 数据过期或缺失时，页面自动降级为观察或暂停。
- 不把交易板块当作行业/概念。

## 9. 2026-07-13 已实现升级

- 首屏新增市场状态带，展示阶段、市场风险预算、研究置信度和数据日期。
- 市场结论包含最强支持证据、最大风险、反证与失效条件，不再只由热度分承担结论。
- 趋势、宽度、流动性、风格、情绪分别展示证据状态；只有当日截面时明确禁止跨期持续性表述。
- 新增偏强、基准、偏弱三情景，各自包含触发、风险动作和证伪条件。
- 页面末端提供证据审计，显示维度状态、依据和缺失字段；过期行情进入“数据暂停”。
