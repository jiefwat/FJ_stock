# 每日大盘模块设计文档

更新时间：2026-07-10  
适用项目：StockTS / A 股个人投研工作台  
模块目标：用最少信息判断今天的市场环境、风险暴露和机会验证方向。

> 边界：只做研究判断，不给确定性收益承诺；所有结论必须带数据日期、来源、条件和失效线。

## 1. 参考结论

市场页的成熟做法不是堆指数，而是把“指数方向、市场宽度、板块主线、异动、日历、数据质量”压成一个行动闸门。

参考：

- [TradingView Markets](https://www.tradingview.com/markets/)
- [TradingView Stock Heatmap](https://www.tradingview.com/heatmap/stock/)
- [TradingView Stock Screener](https://www.tradingview.com/screener/)
- [Yahoo Finance Markets](https://finance.yahoo.com/markets/)
- [CNBC Markets](https://www.cnbc.com/markets/)
- [东方财富数据中心](https://data.eastmoney.com/)

## 2. 分析师标准

每日大盘只回答 4 个问题：

1. **能不能做**：进攻、震荡、防守、暂停。
2. **做什么方向**：强势行业/概念、资金方向、持续性。
3. **用多大风险**：建议现金比例、是否降低仓位、是否只观察。
4. **什么情况失效**：指数破位、主线断裂、跌停扩散、成交不足、数据过期。

必须展示：

| 信息 | 必要字段 | 用途 |
| --- | --- | --- |
| 市场闸门 | 状态、风险等级、目标现金、置信度 | 决定是否行动 |
| 指数脊柱 | 主要指数涨跌、成交、关键位置 | 判断大方向和风格 |
| 市场宽度 | 上涨/下跌家数、涨停/跌停、炸板、成交额 | 判断强弱是否扩散 |
| 板块主线 | 行业/概念强度、持续性、资金、代表股 | 找机会入口 |
| 风险红灯 | 缩量、破位、跌停扩散、重大事件 | 降低行动等级 |
| 数据质量 | 交易日、Provider、刷新时间、缺失字段 | 防止旧数据误判 |

## 3. 输出格式

页面首屏必须输出一张“市场结论卡”：

```text
市场状态：震荡偏强
风险暴露：目标现金 30%-50%，不追高
主线：机器人、算力、消费电子
验证条件：成交额放大，主线前排不破关键均线
失效条件：指数破位，跌停扩散，主线资金净流出
下一步：去“股市机会”验证候选；去“我的持仓”处理高风险仓位
```

禁止输出：

- 只有指数涨跌，没有行动结论。
- 只有板块排名，没有持续性和风险。
- 数据过期时仍给强行动建议。
- 把创业板、科创板、主板当作行业/概念主线。

## 4. 页面设计

页面定位：市场总闸门，不是新闻门户。

```text
┌────────────────────────────────────────────┐
│ 市场结论卡：状态 / 目标现金 / 主线 / 失效线    │
├──────────────┬──────────────┬──────────────┤
│ 指数脊柱      │ 市场宽度      │ 风险红灯      │
├────────────────────────────────────────────┤
│ 板块主线：行业/概念强度 + 资金 + 持续性         │
├──────────────────────┬─────────────────────┤
│ 异动清单              │ 事件日历             │
└──────────────────────┴─────────────────────┘
```

设计要求：

- 首屏只放一个主判断，不放长篇解释。
- 颜色遵守 A 股语义：红强、绿弱、黄观察、灰缺数。
- 板块区必须能跳转到 `股市机会` 并带入主题。
- 风险红灯优先级高于机会提示。
- 移动端顺序：结论卡、风险红灯、板块主线、指数和宽度。

## 5. MVP

必须做：

- 市场状态、目标现金、主线、失效条件。
- 指数脊柱、市场宽度、板块 Top、风险红灯。
- 交易日、Provider、刷新时间、数据质量。
- 跳转：板块到机会，高风险到持仓。

暂不做：

- 复杂宏观解释。
- 长新闻流。
- 盘中秒级刷新。

## 6. 验收标准

- 3 秒内知道今天是进攻、震荡、防守还是暂停。
- 每个市场结论都有证据、条件、失效线。
- 数据降级时，页面自动降级为“只观察”。
- 板块主线只使用行业/概念/题材，不使用交易板兜底。
