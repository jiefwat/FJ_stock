# Aster Market 桌面舒适度与丝滑交互设计

## 决策

本轮采用“工作台精修 + 快照热缓存”，不更换产品架构，不增加登录、交易、手机端或前端框架。目标是让用户每天打开后能在一个首屏内看清市场结论，并在大盘、机会、个股和持仓之间无等待、无跳动地连续工作。

用户已明确授权后续方案默认同意，因此本设计视为批准并直接进入实施计划。

## 现状证据

浏览器逐一操作四个模块后发现：

- 大盘视觉有辨识度，但首屏标题区过高，四维证据落在首屏下方。
- 机会页只有两条测试数据时留白过多，正文与失效条件字号偏小。
- 个股页左侧搜索区长期空置，图表和六维证据占据过多纵向空间。
- 持仓页隐私提示、录入表单、汇总和账本同权，首次使用缺少清晰操作顺序。
- 顶部同时承载品牌、数据时间、四个模块、全局搜索和刷新，内容拥挤但关键动作不突出。
- 公网 84MB 快照每次请求都重新解析；实测首字节约 3.2–4.0 秒，`/api/snapshot` 总耗时约 9.7 秒，本地小快照接口约 1 毫秒。
- 全局搜索每次输入都会立即请求，旧请求不会取消；快速输入时会堆积无效查询。

## 方案比较

### A. 只压缩视觉

只修改 CSS 和排版。风险最低，但接口等待仍会让搜索、个股详情和持仓行情显得迟钝，不能达到“丝滑”。

### B. 工作台精修 + 快照热缓存（采用）

在保留当前 Aster 视觉身份的前提下，收紧首屏、提高字号、优化交互反馈，同时让服务端按快照文件指纹复用解析结果。它能同时解决视觉舒适度、操作连贯性和真实等待时间。

### C. 重型前端 SPA

引入前端构建链和全局状态管理。能力最强，但会显著增加部署和维护复杂度，不符合当前无依赖标准库服务边界。

## 视觉方向

### 主题

产品面向每天复盘 A 股的桌面用户，页面唯一工作是把“环境 -> 机会 -> 个股 -> 持仓”变成连续判断路径。视觉继续借用行情纸带、坐标网格和研究终端标记，但减少展览式空白。

### 设计令牌

- 陶瓷白 `#F3F5F2`：主背景。
- 纸面白 `#FBFCFA`：输入与内容工作面。
- 墨石黑 `#101820`：主文字和结构线。
- 钴蓝 `#1648D8`：导航、主操作和趋势。
- 信号珊瑚 `#F05B42`：风险与需要注意的状态。
- 市场绿 `#177C67`：连接、正向变化和完成反馈。
- 雾灰 `#D8DEDA`：次级分隔线。

标题使用紧凑的窄体无衬线组合，正文继续使用适合中文阅读的 `PingFang SC` / `Noto Sans SC`，数字使用等宽字体。正文最小可读字号从 9–10px 提高到 11–12px，关键说明不再用极小字承担主要信息。

### 标志性元素

保留“市场地平线”，但把它从大幅海报改成可工作的决策跑道：状态、轨迹、指数和涨跌统计在一个紧凑首屏中完成阅读。大胆只放在这一个区域，其他模块保持安静和高可读性。

## 页面结构

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ ASTER  数据新鲜度 | 大盘  机会  个股  持仓 | 全局搜索  / | 刷新          │ 64
├──────────────────────────────────────────────────────────────────────────┤
│ 市场状态 + 核心结论      │ 市场地平线 + 风险点                           │
│ 指数 / 涨跌 / 涨跌停     │ 四维证据摘要                                  │ 首屏
├──────────────────────────────────────────────────────────────────────────┤
│ 当前模块的主要工作面：更大正文、更短标题区、更清晰操作顺序              │
└──────────────────────────────────────────────────────────────────────────┘
```

- 顶部命令带压缩到约 64px；数据时间只显示“交易日 + 更新于 HH:mm”，完整时间放在 `title`。
- 模块名称保持四个，不增加侧栏；数字键 `1–4` 切换模块，`/` 聚焦全局搜索。
- 模块切换记住各自滚动位置；不再每次强制平滑滚回顶部。
- 模块标题区从约 176px 压缩到 96–112px，重要内容尽量进入首屏。
- 所有可点元素拥有一致的悬停、按下、键盘焦点和禁用状态。

## 模块调整

### 大盘分析

- 市场状态字号从海报级缩到工作台级，结论控制在两行内。
- 指数、上涨/下跌、涨停/跌停并入首屏状态区。
- 四维证据紧接地平线，不再被大面积留白推到第二屏。
- “下一步”保留，但作为紧凑观察清单，不与主结论竞争。

### 市场机会

- 机会标题区压缩，机会行正文提高到 11–12px。
- 主题、阶段、强度、候选和失效条件保持横向走廊，但行高由展示型改为工作型。
- 候选按钮增加明确悬停和按下反馈；点击后立即切换个股模块并显示骨架，不等待接口完成才反馈。

### 股票分析

- 左侧搜索轨道保持固定宽度和粘性，搜索结果可连续浏览。
- 当前股票标题、价格、趋势和本机持仓动作放在一条紧凑摘要栏。
- 图表高度降低，六维证据提高可读字号；证据、风险、事件在首屏下半部开始出现。
- 搜索输入使用共享防抖、`AbortController` 取消旧请求，并缓存本次会话搜索和个股详情。
- 加载状态使用结构化骨架，不用孤立的“正在读取…”文字。

### 我的持仓

- 顺序固定为：录入动作 -> 组合汇总 -> 暴露 -> 持仓账本。
- 隐私提示收敛成一条轻量状态说明。
- 保存、编辑、删除和清空提供即时文字反馈；清空仍只影响浏览器本地数据。
- 获取多只股票时复用个股缓存，避免同一代码重复请求。

## 动效与快捷操作

- 模块切换：`160–180ms` 淡入与 8px 位移，只对新模块执行一次。
- 按钮：按下时轻微下沉 1px；悬停只改变底色和边框，不使用大面积发光。
- 数据加载：骨架使用低对比度横向扫光；请求完成后内容淡入。
- 成功反馈：页面右下角使用单一短提示，约 1.8 秒自动消失。
- `prefers-reduced-motion` 下取消位移、扫光和自动平滑滚动。
- 键盘在输入框中输入时不触发模块快捷键；`Escape` 清空搜索或关闭当前提示。

## 快照热缓存

服务端为每个 `create_server` 实例创建线程安全的快照缓存：

1. 请求到达时对配置的快照路径执行 `stat()`。
2. 指纹由目标文件的 `st_ino`、`st_size` 和 `st_mtime_ns` 组成；符号链接目标被原子替换时指纹变化。
3. 指纹未变化时复用同一个 `SnapshotResult` 和页面 `view`，不重复读取 84MB JSON。
4. 指纹变化时只允许一个线程重新解析，成功后原子替换缓存。
5. 文件缺失或损坏继续返回现有 503/中断状态，不使用旧数据伪装成功。

动态响应继续使用 `Cache-Control: no-store`；这是服务端解析缓存，不是浏览器行情缓存。部署后目标是热缓存状态下首页和个股 API 首字节低于 250ms。

## 错误与状态

- 搜索请求被新输入取消：不显示错误。
- 接口失败：保留当前内容，在工作面内显示具体原因和“重新读取”动作。
- 快照更新中：旧缓存不伪装为新数据；下一次成功解析后展示新时间。
- 持仓行情部分失败：保留本机代码、数量和成本，单行标注行情不可用。
- 页面刷新：按钮进入“读取中”状态，防止重复点击；浏览器仍执行完整重载。

## 测试与验收

- 缓存测试证明相同指纹只解析一次，原子替换文件后重新解析。
- 并发请求不会重复解析同一份快照。
- 缓存不改变 404、503、安全头或 `no-store` 契约。
- 资源测试覆盖快捷键、滚动位置、请求取消、会话缓存、骨架和 reduced-motion。
- 真实浏览器完成四模块切换、机会到个股、搜索、加入持仓和持仓保存链路。
- 1280、1536、1920 三档桌面无横向溢出；不新增 viewport meta、手机导航或 `@media (max-width: ...)`。
- 公网热缓存下首页、市场分析、机会和个股接口首字节均低于 250ms。
- 主服务、Signal Desk 和 Nginx 保持健康，公网仍只开放只读 GET。
