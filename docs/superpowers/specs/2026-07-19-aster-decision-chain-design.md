# Aster Market 今日研判链设计

## 决策

本轮采用“全局研判链 + 四个递进工作区”。保留公开只读、桌面端、原生 JavaScript 和浏览器本地持仓边界，但把产品叙事从四个并列模块改成一条分析师工作流：

`市场环境 -> 参与许可 -> 主线判定 -> 个股验证 -> 持仓检查`

用户已明确后续方案默认同意，因此本设计直接视为批准并进入实施计划。

## 现状问题

当前页面的问题不是信息不足，而是判断层级缺失：

- 大盘、主题、候选、事件和持仓各自成立，但没有一个结论把它们串起来。
- 生产快照处于“收缩 / 风险升高”，上涨参与度仅 8.8%，页面仍把单股异动排列成“市场机会”，分析语义相互冲突。
- 主题强度只表达排序，没有回答“这是确认主线、候选主线，还是逆势异动”。
- 个股分析不知道自己在验证哪条主线；持仓也不知道当前市场许可是什么。
- 用户需要自己从多个区块拼结论，页面没有承担分析师应做的归纳工作。

## 方案比较

### A. 单页长报告

把全部内容变成一条纵向报告。主线最直接，但查股票和管理持仓会被迫反复滚动，不适合高频桌面使用。

### B. 全局研判链 + 递进工作区（采用）

顶部始终显示市场许可与主线结论；四个工作区依次负责研判、主线扫描、个股验证和持仓检查。它既建立连续推理，也保留工具效率。

### C. 高密度行情仪表盘

增加更多数字、排行和图表。信息密度最高，但会继续放大“块很多、结论很少”的问题。

## 分析师规则

### 1. 市场参与许可

市场参与许可只表达观察强度，不提供交易指令：

- `扩张` 且风险 `可控`：`主动跟踪`，允许寻找已确认主线。
- `轮动` 且风险不为 `升高`：`结构确认`，只寻找候选主线并等待扩散。
- `收缩` 或风险 `升高`：`防守等待`，所有强势主题降级为逆势异动。

### 2. 主线判定

主题必须同时经过市场环境和自身证据两层过滤：

- `确认主线`：市场许可为主动跟踪，主题非高分歧，上涨参与度不低于 70%，连续性不少于 2 日。
- `候选主线`：市场许可为结构确认，主题非高分歧，阶段为扩散或加速。
- `逆势异动`：市场许可为防守等待，即使涨幅居前也不得称为主线。
- `方向分歧`：市场允许观察，但第一主题处于高分歧或缺少连续性。
- `未形成主线`：没有可用主题证据。

页面必须同时展示主题、阶段、强度、参与度、连续性、候选标的和失效条件，不能只用一个涨幅排序替代判断。

### 3. 今日结论

一句完整结论必须回答四件事：

1. 当前市场环境是什么。
2. 当前允许采用什么观察强度。
3. 是否存在确认主线；若没有，最强方向属于什么性质。
4. 下一次需要看到什么变化才能升级判断。

生产快照的预期表达为：`市场收缩，当前以防守等待为主；尚未形成可确认主线，人工智能属于逆势异动。等待跌停压力回落并确认上涨参与度扩散。`

## 信息架构

四个导航名称改为真实的分析顺序：

1. `今日研判`：给出结论、许可、主线状态和升级条件。
2. `主线扫描`：比较八个主题的证据与失效条件。
3. `个股验证`：验证主线映射股票的趋势、动量、波动和事件。
4. `持仓检查`：把当前市场许可和主线状态带入本机持仓审视。

命令带下方增加全局研判链。切换任何工作区时，用户都能看到当前环境、参与许可和主线结论，不需要回到第一页重新寻找上下文。

## 今日研判页面

用户追加要求页面只保留精华，因此首页严格限制为四层：研判链、今日结论、指数与四维证据、主线梯队。事件流、全市场候选流、大幅轨迹图、英文装饰标题和重复解释段落全部移除。

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ ASTER | 数据 | 1 今日研判 2 主线扫描 3 个股验证 4 持仓检查 | 搜索       │
├──────────────────────────────────────────────────────────────────────────┤
│ 01 市场环境 ─ 02 参与许可 ─ 03 主线判定 ─ 04 验证对象 ─ 05 升级条件     │
├───────────────────────────────┬──────────────────────────────────────────┤
│ 今日结论                      │ 四条核心证据                            │
│ 市场收缩，防守等待            │ 指数 / 广度 / 跌停 / 主题分歧            │
│ 尚未形成可确认主线            │ 当前最强方向及其降级原因                  │
├───────────────────────────────┴──────────────────────────────────────────┤
│ 主线梯队：状态 | 主题 | 参与度 | 连续性 | 领涨映射 | 失效条件            │
└──────────────────────────────────────────────────────────────────────────┘
```

原来的市场地平线、事件流和全市场候选流从首页删除。视觉焦点只保留“研判链”，这条连续线是页面唯一的标志性元素；升级条件直接附在今日结论下，不再单独形成解释模块。

## 视觉系统

保留现有陶瓷白、墨黑、钴蓝、信号橙和市场绿，避免又一次无意义换肤：

- 陶瓷白 `#F3F5F2`：工作面。
- 墨石黑 `#101820`：结论和结构。
- 钴蓝 `#1648D8`：当前步骤与可验证方向。
- 信号橙 `#F05B42`：防守、失效和高风险。
- 市场绿 `#177C67`：确认和扩散。
- 雾灰 `#D8DEDA`：证据分隔。

标题继续使用紧凑窄体，正文使用中文无衬线，数据使用等宽字体。数字序号在这里代表真实推理顺序，不是装饰。

## 交互

- `1` 至 `4` 继续切换工作区，但 accessible name 使用完整工作区名称。
- 点击主线候选股票进入个股验证，并保留该主题作为当前验证上下文。
- 全局搜索仍用 `/` 聚焦，`Escape` 清空；搜索结果进入个股验证。
- 模块滚动位置、请求取消、会话缓存和 toast 保持。
- 动效只用于研判链首次连线和工作区 180ms 入场；reduced motion 下全部关闭。

## 错误与边界

- 没有主题时明确显示“未形成主线”，不使用示例主题补位。
- 收缩市场中的高涨幅主题必须标记为逆势异动，不得显示为确认主线。
- 缺失参与度、连续性或分歧字段时降级为方向分歧。
- 个股与持仓继续显示数据质量，不生成目标价、仓位比例或买卖指令。
- 持仓数量与成本仍只存在 `aster.portfolio.v1`，不进入网络请求。
- 不新增 viewport meta、手机导航或窄屏媒体查询。

## 测试与验收

- 单元测试覆盖扩张、轮动、收缩和无主题四种决策简报。
- Presenter 测试证明 `/api/snapshot` 包含 `decision_brief`。
- UI 测试覆盖全局研判链、四个递进名称、主线梯队和防守状态。
- 真实浏览器验证生产收缩快照不会把人工智能写成确认主线。
- 1280、1536、1920 三档桌面无横向溢出。
- 搜索、机会到个股、加入持仓和删除链路保持可用。
