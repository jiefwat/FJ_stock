# 可用研究决策系统设计

## 背景

四大原生研究工作台已经完成服务端能力接入、供应商中性协议和极简页面，但当前归一化层按上游字段顺序取每行前四项。真实响应通常先返回股票代码、简称、最新价和涨跌幅，财务、经营、预期和事件字段排在后面。因此页面虽然显示“财务质量”“经营结构”“机构预期”等卡片，内容却重复报价信息，不能支持研究判断。

这不是页面文案问题，而是产品协议把“返回了数据”误判成“返回了该能力所需证据”。新版本要把连通的接口升级成可用于复核决策的研究系统。

## 目标

- 每个能力只展示与该研究维度相关的事实，不再以字段顺序决定内容。
- 股票代码、名称、最新价和涨跌幅只作为身份或行情上下文，不冒充财务、经营、预期和事件证据。
- 只有命中能力所需语义字段的结果才算成功；身份字段或空泛文本必须进入缺口。
- 个股首屏从真实事实形成“财务方向、现金流质量、经营结构、预期变化、事件风险”判断，不重复同一组事实。
- 市场和机会模块使用经真实接口校准的确定性查询，优先获得指数、板块和候选的可比较字段。
- 页面仍保持三秒可读：一条判断、一条动作、一个最大风险、最多三条核心发现；完整原始依据放入折叠区。
- 保留现有登录、限流、请求大小、超时、响应上限、凭证隔离、供应商中性和持仓隐私边界。

## 方案比较

### 方案 A：继续通用截取，扩大字段数量

把每行四个字段扩大到八个或十二个，改动最小，但身份字段仍会占据主要位置，页面噪声上升，上游字段顺序变化后问题会再次出现。不采用。

### 方案 B：让模型对原始响应做自由总结

语言表达自然，但会增加延迟、成本和不可复现性，也可能在证据不足时补全不存在的判断。与当前确定性、可审计边界冲突，不采用。

### 方案 C：能力级确定性提取与质量门禁

为每个能力定义身份字段、核心字段模式、字段优先级、格式化规则和最低证据要求。归一化层先剔除身份与内部元数据，再按能力选取研究字段；未达到最低要求就标记缺失。结论层只使用通过门禁的事实，并做去重和跨期比较。采用。

## 架构

```text
固定能力查询
  -> 上游动态表格
  -> 能力级字段分类
     -> identity / quote / evidence / internal
  -> 专业值格式化
  -> 最低证据质量门禁
  -> 事实行去重与跨期压缩
  -> 模块判断、动作、风险、三条发现
  -> 供应商中性产品协议
```

`src/stock_ts/research_evidence.py` 负责动态字段到研究证据的转换，避免继续扩大 `research_engine.py` 的职责：

- `CapabilitySchema`：能力所需字段模式、身份/报价排除模式、字段优先级和最低证据数。
- `normalize_capability_rows(capability, raw)`：提取、格式化、去重并返回该能力的证据行。
- `has_semantic_evidence(capability, rows)`：判断结果是否达到该能力的最低语义要求。
- 值格式化只处理可确定的类型：金额、百分比、日期、列表和普通文本；不猜测单位，不补全缺失值。

`src/stock_ts/research_engine.py` 继续负责固定查询、并发、缓存和模块结果编排，但改为把能力名称传给证据归一化器，并根据语义证据而不是非空行判断成功。

## 能力证据规则

### 公共排除字段

以下字段默认不计入研究证据：股票代码、证券代码、股票简称、证券简称、最新价、现价、涨跌幅、市场、交易所、内部 trace、能力 id、供应商、授权、网关和来源地址。

报价字段可以在 `market` 能力中作为行情证据，但不能在 `finance`、`business`、`consensus` 或 `event` 中单独使能力通过门禁。

### 个股能力

| 能力 | 优先证据 | 最低要求 | 首屏表达 |
| --- | --- | --- | --- |
| 财务 | 营业收入、归母净利润、经营现金流、ROE、毛利率、负债 | 至少一个财务字段 | 增长方向与现金流匹配情况 |
| 经营 | 主营产品、业务范围、客户、供应商、竞争对手、市场地位 | 至少一个经营字段 | 收入来源、竞争位置与集中风险 |
| 机构预期 | 未来年度净利润/营收预测、评级、目标价、上调/下修 | 至少一个预期字段 | 未来两年预期轨迹与变化方向 |
| 事件 | 业绩预告/快报、公告、解禁、质押、监管、诉讼、增减持 | 至少一个事件字段 | 最新变化、潜在催化和反证 |

个股结论不得只写“证据已更新”。应按真实事实生成可核查表达，例如：

- 财务：连续年度收入与利润同向改善，但经营现金流未同步，结论为“增长质量仍需现金流确认”。
- 预期：2026、2027 预测利润上升，结论为“机构预期仍在增长轨道”；只有单期数据时不声称趋势。
- 事件：最新季度利润同比下降，优先进入最大风险；没有负面字段时保留通用风险边界。

### 市场、持仓与机会能力

- 市场：指数能力必须包含指数名称与点位/涨跌/成交之一；板块能力必须包含板块名称与强度、成交额、排名或热度之一。
- 持仓：每只股票独立通过事件、公告、预期和行情门禁；首屏每只持仓最多一条且优先展示风险等级更高的事实。
- 机会：板块筛选必须返回行业或概念名称；A 股筛选必须返回股票身份以及盈利、成交、排名或筛选理由之一；只有代码、名称和报价时视为缺失。
- 新闻与公告搜索结果使用标题、日期、摘要和文档类型；链接只保留安全的公开 URL，不进入首屏摘要。

## 值格式化

- 明确以元计价的金额：绝对值大于等于一亿显示为 `12.35 亿`，大于等于一万显示为 `8560 万`；保留原字段名中的报告期。
- 明确比例字段：数值已经带 `%` 时保留；字段语义为比例且值为小数时转为百分比；不确定单位时保持原值。
- 日期：`YYYYMMDD` 和可识别时间统一为 `YYYY-MM-DD`，不改变仅包含年份或季度的报告期。
- 多期字段：同一指标按报告期从新到旧保留最多三期，并生成紧凑摘要；完整字段仍在折叠依据中可见。
- 列表和长文本：首屏截取最重要的三项，完整依据限制长度并保留省略提示。

## 去重与排序

- 事实指纹由规范化后的字段名和值组成；完全相同或只差身份字段的卡片只保留一张。
- 首屏优先级：明确风险变化 > 跨期经营变化 > 预期变化 > 静态业务描述 > 单日行情。
- 个股最多三条发现，尽量覆盖三个不同能力；同一能力只有在存在独立高风险事实时占用第二张卡。
- 持仓最多三条发现，每只持仓最多一条；同一持仓多维度均有结果时优先事件、预期、行情。

## 查询校准

查询使用短、可解析、与能力返回模式一致的条件表达，避免把自然语言研究问题直接当作选股条件：

- 指数：明确列出上证指数、深证成指、创业板指及最新点位、涨跌幅和成交额。
- 板块：使用“行业板块，按成交额或热度排序，前 5”一类明确筛选和排序条件。
- A 股候选：使用可被选股能力解析的财务和成交条件，并限制前 10；若实时校准仍无结果，页面明确显示候选能力缺失，不用新闻或事件结果冒充候选。
- 宏观、新闻和公告继续作为事件证据，不承担指数或候选替代职责。

查询校准必须通过真实接口记录返回行数和字段类型，但自动化测试只使用固定样例，避免日常测试依赖网络。

## 页面设计

现有“判断带”保留，但研究信息分成两个阅读速度：

```text
当前判断                 证据完整度 3/4
一句股票或市场专属判断

现在怎么做               最大风险

01 财务方向  证据标签
一句结论
营业收入  26.4 亿  ->  31.8 亿
归母净利   1.2 亿  ->   1.6 亿

02 经营与竞争
一句结论 + 最多两项关键事实

03 预期或事件
一句结论 + 最多两项关键事实

[完整依据与缺口] [重新分析]
```

- 首屏卡片标题使用结论名称，不重复“股票名 · 能力名”。股票名只在页面标题或持仓标签出现一次。
- 卡片摘要必须是对事实的压缩判断；事实列表保留数值和报告期，避免把摘要再复制一遍。
- 增加 `已确认维度/总维度`，让“partial”有可理解的含义。
- 缺失维度集中显示在完整依据入口旁，不单独占据大卡。
- 移动端保持单列，判断、动作、风险和第一条发现应在约两个屏幕内读完。

## 失败与降级

- 请求异常：能力状态为 `failed`，模块可部分成功。
- 有返回但只有身份/报价：能力状态为 `insufficient`，计入缺口，不显示空泛卡片。
- 有研究字段但不足以形成趋势：展示事实，结论明确为“仅单期，趋势待确认”。
- 全部能力无语义证据：暂停判断，不回退旧规则、不拼接新闻冒充结论。
- 上游字段未知：放入完整依据的“未分类字段”不进入首屏；只有在添加明确语义规则和测试后才能升级为核心证据。

## 安全边界

- API key 继续只从服务端环境读取，不进入 HTML、JSON、日志、测试样例或提交。
- 浏览器不能指定能力 id、版本、网关或查询模板。
- 保留公网登录、本地匿名显式开关、用户/IP 限流、16 KiB 请求限制、12 秒能力超时和 1 MiB 响应限制。
- 持仓只传最多三只股票的代码和名称，不传数量、成本、权重、备注、账号和 Cookie。
- 页面与产品 JSON 不出现供应商品牌、内部能力标识、trace 或授权字段。
- 输出是条件化研究复核，不自动交易，不承诺收益。

## 验收标准

- `603278` 个股页不再在财务、经营、预期和事件卡片重复代码、名称、价格和涨跌幅。
- `603278` 财务依据可展示真实多期收入、利润、经营现金流和 ROE；经营依据可展示主营、业务范围或竞争关系；预期依据可展示未来年度预测；事件依据可展示最新业绩变化。
- 只有身份或报价字段的能力被标记为缺失，不把模块误报为完整。
- 首屏卡片无重复指纹，最多三条，标题与摘要不重复股票名和能力名。
- 市场和机会真实调用记录各能力的返回行数；零行能力使用校准查询后重新验证。
- 四模块 API 和页面不含供应商术语、内部元数据或凭证。
- 现有安全、鉴权、限流、请求大小、超时、响应上限和持仓隐私测试继续通过。
- 桌面 1280px 与移动 390px 无横向溢出，页面在真实数据下可读。
