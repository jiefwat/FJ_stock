# 多周期、数据闸门优先的研究系统设计

## 目标

修复个股、热点机会和每日大盘把单日价格表现误当成可持续判断的问题。系统必须先判断数据是否足以支持结论，再使用 3、5、10、20 个交易日的价格、量能、回撤和上涨分布形成条件化判断。

统一研究顺序：

```text
数据是否可用 -> 多周期处于什么阶段 -> 支持与反证 -> 确认条件 -> 失效条件
```

外部研究只补充财务、经营、机构预期、行业、事件、公告和研报证据，不接管本地数据闸门、持续性阶段、机会排序或风险结论。

## 已确认问题

线上 `300725 药石科技` 可以稳定复现以下错误：

- 大盘交易日为 `2026-07-15`，股票 K 线只到 `2026-07-14`。
- 股票有 120 根日线，但财务、估值、资金流、新闻、公告和行业主题均缺失。
- 外部八项研究全部不可用，页面进入本地降级。
- 本地规则仍按价格和均线输出“谨慎进攻”，并把七个缺失维度铺成重复的“待补”卡。
- 候选评分把缺失资金流当作中性资金并获得正分，且没有在产品结果中解释多日持续性。
- 每日大盘的异动名单按单日绝对涨跌幅生成，不能代表后续持续性。

问题根因是数据完整度、数据新鲜度和方向判断没有形成硬边界，不是页面样式不足。

## 方案选择

### 方案 A：本地多周期判断为主，外部证据补强（采用）

本地快照负责可重复的行情新鲜度、多周期价格阶段、持续性评分和风险闸门；外部研究只填补非行情维度。页面消费同一份供应商中性的产品协议。

优点：额度、网络或外部接口异常时仍能给出诚实判断；每个结论可由本地日线复算；不会因外部文案覆盖风险闸门。缺点：需要新增多周期统计模型并更新快照兼容判断。

### 方案 B：外部研究优先决定结论

信息维度更丰富，但额度耗尽时会再次退化，且难以保证结论与本地行情日期一致，不采用。

### 方案 C：只调整页面文案

不能修复错误加分、陈旧数据和单日脉冲排序，不采用。

## 研究领域模型

新增供应商中性的多周期统计结果，建议放在 `professional_analytics.py`：

```text
MultiHorizonProfile
  as_of
  bar_count
  return_3d / return_5d / return_10d / return_20d
  up_days_5d / up_days_10d
  volume_ratio_5d_to_20d
  drawdown_10d
  distance_to_20d_high
  ma_alignment
  stale
  price_reliable

ContinuationAssessment
  score
  stage
  confidence
  support
  counter_evidence
  confirmation
  invalidation
```

阶段固定为：

- `延续观察`：5 日和 10 日趋势同向，最近上涨不是集中于单日，均线结构和回撤可接受。
- `突破待确认`：刚突破短期区间，但持续天数、主题或量能仍需下一交易日确认。
- `反弹待验证`：短期转强，但 20 日结构仍弱或距前高压力较近。
- `脉冲待验证`：单日大涨贡献了大部分 5 日收益，缺少此前趋势基础。
- `过热回避`：短期涨幅、均线偏离、波动或回撤风险过高。
- `剔除`：行情过期、真实 K 线不足、价格不可靠或明确风险闸门触发。

## 数据正确性闸门

### 新鲜度

股票和候选的最新日线日期与大盘交易日比较：

- 同日：可用于当日判断。
- 落后一个交易日：只允许历史复盘，方向结论降级为“等待数据更新”。
- 落后两个及以上交易日：阻断机会排序和方向性结论。

流水线刷新状态必须反映最终数据日期。请求成功但必需股票未到预期交易日时，`kline_refresh` 只能是 `partial` 或 `stale`，不能标为 `ok`。

### 完整度

个股八个产品维度为财务质量、经营结构、机构预期、事件风险、行情资金、行业位置、公告事项和研报观点。

- 行情过期：无论其他维度多少，动作不得高于“等待确认”。
- 有效维度少于 4 项：固定为“数据不足”，不得输出进攻类判断。
- 只有行情可用：展示多周期行情事实和关键缺口，不渲染七张重复空卡。
- 外部证据补齐后重新计算覆盖度，但外部内容不能解除行情过期闸门。
- 资金流缺失记为未知，不得获得中性或正向分数。

### 真实 K 线

持续性判断至少需要 20 根可靠日线。行情截面推导出的单日占位 K 线不能进入多周期评分。`price_reliable=false`、根数不足或日期异常时直接进入 `剔除` 或“数据不足”。

## 多周期持续性评分

总分 100，只作为研究排序，不映射为收益概率：

| 维度 | 上限 | 说明 |
| --- | ---: | --- |
| 5/10/20 日趋势一致性 | 25 | 多周期同向优于单日上涨 |
| 上涨日分布 | 15 | 最近 5/10 日上涨天数，防止收益集中在一天 |
| 均线结构 | 15 | 5、10、20 日均线排列与价格位置 |
| 量能质量 | 10 | 5 日均量相对 20 日均量，放量但不过度失真 |
| 回撤与波动 | 15 | 近 10 日最大回撤和收益波动 |
| 主题与市场环境 | 10 | 主题持续性和市场风险预算 |
| 资金、事件、基本面 | 10 | 只在真实证据存在时计分 |

硬扣分：

- 单日涨幅贡献超过 5 日涨幅的 70%，且此前四日没有连续性：至少扣 18 分，阶段为“脉冲待验证”。
- 5 日涨幅过高且距 20 日均线偏离过大：至少扣 15 分，阶段为“过热回避”。
- 近 10 日最大回撤过大：按风险等级扣分。
- 主题未识别、资金缺失、事件缺失分别记为未知，不以默认值补分。
- 行情过期或不可靠：不进入排序。

阈值使用测试固定，并在实现时集中定义为常量，禁止散落在页面层。

## 四模块信息架构

### 每日大盘

固定顺序：

1. 市场状态与风险预算。
2. 真实涨跌宽度和涨跌停分布。
3. 主题持续性，而不是只看当日主题涨幅。
4. `持续性观察名单`：最近 5/10/20 日表现、上涨天数、阶段、持续理由、确认条件和失效条件。
5. `单日异动监控`：按单日绝对涨跌排序，只用于发现异常，不作为推荐名单。

持续性名单最多 8 只，默认展示前 5 只。没有候选通过闸门时明确显示“当前无满足多周期条件的股票”，不回退到当日涨幅榜。

### 我的持仓

固定顺序：

1. 组合风险与优先处理顺序。
2. 每只持仓的多周期阶段、5/10/20 日表现和数据日期。
3. 当前动作、主要原因、确认条件和失效条件。
4. 主题集中与同主题分化。
5. 个股分析入口。

持仓动作仍可结合用户成本和仓位，但多周期价格判断必须来自与个股页相同的统计函数，避免两个页面给出不同趋势。

### 个股分析

固定顺序：

1. 数据闸门：日期、可靠性、有效维度和阻断原因。
2. 整体结论：数据不足、等待确认、风险优先、延续观察或反弹待验证。
3. 多周期价格结构：3/5/10/20 日表现、上涨天数、回撤、量能和均线。
4. 支持证据、主要反证、确认条件和失效条件。
5. 八维研究：只展开有真实事实的维度；缺失维度合并为一张“关键缺口”卡。
6. 完整依据。

数据不足时不输出方向性动作，只提供恢复数据和下一步验证动作。

### 热点机会

固定顺序：

1. 机会闸门：当前是否允许筛选。
2. 多周期候选列表，按持续性评分排序。
3. 每只候选的阶段、5/10/20 日表现、上涨日分布、最大回撤和量能质量。
4. 入选原因、主要风险、确认条件和失效条件。
5. 主题上下文与风险排除。

单日上涨只是一项事实，不作为主要入选原因。列表不得使用“今日涨幅靠前”替代持续性判断。

## 页面与协议

继续使用 `ResearchWorkspaceResult`、`ResearchModuleSection` 和 `ResearchModuleItem`。新增分区：

```text
stock-data-gate
stock-multi-horizon
market-continuation
market-movers
opportunity-candidates
```

候选和持续性股票的事实字段固定包含：

```text
阶段判断 / 持续性评分 / 5日表现 / 10日表现 / 20日表现 /
上涨天数 / 最大回撤 / 入选原因 / 确认条件 / 失效条件
```

页面保持现有金融终端视觉，不重做导航和交互逻辑。动态渲染继续使用 DOM API 与 `textContent`，不使用外部响应拼接 `innerHTML`。产品协议和页面不得出现供应商品牌、能力标识、调用链字段或凭证。

## 快照兼容

大盘快照必须同时包含 `market-pulse`、`market-continuation` 和 `market-movers`。机会快照的候选必须包含阶段、多周期表现、入选原因、确认条件和失效条件。

旧快照缺少任一新合同字段时视为不兼容，使用当前本地数据重新生成；不得继续展示旧的单日候选协议。

## 失败与降级

| 场景 | 行为 |
| --- | --- |
| 外部研究额度或网络不可用 | 保留本地多周期分析，不展示底层错误 |
| 股票行情落后市场一天 | 历史复盘可见，动作降级为等待数据更新 |
| 股票行情落后两天以上 | 阻断方向判断和机会排序 |
| 只有行情、其余维度缺失 | 展示多周期行情和一张关键缺口卡 |
| 不足 20 根真实 K 线 | 不计算持续性评分 |
| 无候选通过闸门 | 显示空状态，不用单日涨幅榜补位 |
| 外部事实与本地风险闸门冲突 | 保留外部事实，服从本地风险闸门 |

## 测试策略

按 TDD 实施，先增加失败测试：

1. 个股行情日期落后大盘时不能输出“谨慎进攻”。
2. 个股只有行情一项可用时显示数据不足，并把七项缺口合并。
3. 缺失资金流不给候选加分。
4. 单日大涨、此前四日走弱的股票进入“脉冲待验证”，排名低于多日平稳上行股票。
5. 连续上行但过度偏离均线的股票进入“过热回避”。
6. 行情过期、少于 20 根 K 线或价格不可靠的股票不能进入机会与持续性名单。
7. 每日大盘同时包含持续性名单和异动监控，二者候选逻辑独立。
8. 机会候选包含固定的多周期事实、确认条件和失效条件。
9. 旧大盘和机会快照因缺少新合同自动重建。
10. 动态页面不出现供应商品牌或内部字段，桌面和 390px 无横向溢出。

完成后运行：

- 多周期统计、个股兜底、机会和快照专项测试。
- Web 认证、持仓编辑、四工作台接口和 systemd 合同测试。
- `make lint`、全量 `make test` 并与既有失败基线比较。
- 本地桌面与 390px 浏览器检查。
- 公网登录态四模块冒烟、服务、三个既有定时器和错误日志检查。

## 验收标准

1. 截图中的药石科技场景不再输出进攻类结论，不再展示七张重复待补卡。
2. 数据日期落后、价格不可靠和证据覆盖不足均能阻断方向性判断。
3. 热点机会不按单日上涨生成主要判断，候选明确展示 5/10/20 日证据和阶段。
4. 单日脉冲、过热、反弹和多日延续能由固定规则区分。
5. 每日大盘新增最多 8 只持续性观察股票，并与单日异动名单分开。
6. 持仓、个股和机会复用同一多周期统计函数，不出现跨页面趋势冲突。
7. 外部研究不可用时系统仍可用，但不伪造财务、资金、事件或持续性结论。
8. 新快照合同、认证、持仓隔离、移动端和敏感字段过滤不回归。
9. 部署后公网健康检查、登录态四模块、现有服务与三个定时器均正常。
