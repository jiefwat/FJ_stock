# Aster Market 全新股票软件设计

## 决策

旧 StockTs 产品、代码、页面、模块、文案和项目记忆全部作废。Aster Market 从无父提交的孤儿分支开始，不继承旧 Git 历史，也不复用旧组件。

## 产品定位

Aster Market 是公开只读的 A 股市场态势软件。它不做自动交易、不管理个人账户、不展示私人持仓，也不输出确定性买卖建议。首页必须在一个桌面工作面内回答四个问题：

1. 市场处于什么状态？
2. 强度集中在哪里？
3. 哪些股票值得继续观察？
4. 今天最需要防范什么？

## 方案比较

### 方案 A：传统行情终端

多列表格、黑色背景、红绿闪烁。信息密度高，但与大量券商软件同质化，也容易变成装饰性 KPI 墙。

### 方案 B：卡片仪表盘

白底卡片、侧栏、指标块。实现容易，但与旧 StockTs 和常见模板过于相似，明确排除。

### 方案 C：市场地形台（采用）

无侧栏。顶部是一条横向命令带，中部用“市场地平线”把风险和机会组织成一个连续视觉面，底部是候选流和事件流，右侧用窄时间轴表达下一步。结构像航空气象台，而不是研究报告或券商行情列表。

用户已明确授予全部产品与技术决策权，因此方案 C 直接视为批准。

- 主文字“墨石” `#101820`

- 风险“信号橙” `#FF5A36`
- 正向“海藻绿” `#16856B`

标题使用 `Arial Narrow / Noto Sans SC` 的压缩无衬线组合；正文使用 `Avenir Next / Noto Sans SC`；数据使用 `IBM Plex Mono / SFMono-Regular`。不使用宋体、黄铜金、深色侧栏、序号导航或旧项目配色。

标志性元素是贯穿页面的“市场地平线”：一条根据涨跌家数、风险和板块强度改变形状与颜色的 SVG 轨迹。它只编码市场状态，不作为纯装饰。

## 功能范围

### 市场

- 上证、深证、创业板指数快照
- 上涨/下跌家数、涨停/跌停、北向流量
- 根据广度和风险生成简洁市场状态，不生成买卖结论

### 发现

- 主题强度按涨幅、上涨占比、成交变化和连续性排序
- 候选股票展示代码、名称、主题、涨跌幅、观察原因和风险提示
- 所有候选明确标注“观察，不是买点”

### 事件

- 展示带来源和时间的市场新闻
- 不抓取、不存储个人信息

### 数据状态

- 展示快照生成时间、交易日和来源
- 数据文件缺失、过期或字段不完整时显示明确降级状态
- 不用示例数据冒充实时行情

## 技术架构

项目类型为 `api-service`，治理等级为 `standard`，Python 3.11+，只使用标准库。

- `src/aster_market/models.py`：不可变领域模型
- `src/aster_market/snapshot.py`：只读 JSON 快照适配器和降级逻辑
- `src/aster_market/presenter.py`：市场状态、主题和候选排序
- `src/aster_market/ui.py`：HTML 文档与 JSON 序列化
- `src/aster_market/web.py`：HTTP 路由、缓存边界和启动入口
- `src/aster_market/assets/`：独立 CSS 与 JavaScript

公开接口：

- `GET /`：桌面软件
- `GET /healthz`：纯文本 `ok`
- `GET /api/snapshot`：只读、供应商中立的页面数据

配置仅使用环境变量：`HOST`、`PORT`、`ASTER_SNAPSHOT_PATH`。没有账户、数据库、通知、AI、问财或交易接口。

## 数据流

服务器从 `ASTER_SNAPSHOT_PATH` 读取 JSON；默认路径为 `data/market_snapshot.json`。生产部署时可以把现有行情文件复制到这个新路径，但新代码只依赖文档化的字段契约，不依赖旧 Python 实现。

解析失败时返回 `status=unavailable` 和缺失原因；页面仍可打开，但不会渲染虚构行情。

## 测试与验收

- 模型、解析、状态推导、HTML 转义、HTTP 路由和缓存头均有 pytest 覆盖

- 本地 1280、1536、1920 三档桌面验证无横向溢出
- 浏览器控制台无页面错误
- 服务器部署后 `/healthz` 与 `/api/snapshot` 返回 200

- 不修改 Signal Desk、Nginx、timer、凭证或原始行情文件
