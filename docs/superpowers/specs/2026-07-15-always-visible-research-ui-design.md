# 始终有内容的研究工作台设计

## 目标

四个核心工作台不能因为外部研究服务额度、超时或网络错误而变成空页面。外部研究仍是首选证据源；不可用时，系统使用当天本地行情、财务、资金、公告、持仓和候选快照生成同一产品协议下的条件化结论。

统一阅读顺序保持不变：

```text
数据状态 -> 核心结论 -> 现在怎么做 -> 最大风险 -> 三条依据 -> 模块内容 -> 完整依据
```

## 当前问题

`ResearchSnapshotStore` 只允许保存 `market` 和 `opportunity`。`portfolio` 与 `stock` 在实时研究返回 `unavailable` 时没有第二条证据路径，`deliver_research` 直接返回空结果。前端随后隐藏空的 `module_sections` 与 `module_items`，最终只剩“研究暂不可用”。

这不是排版问题，而是交付链缺少可用数据时的产品级兜底。

## 方案选择

### 方案 A：外部研究优先，本地证据兜底（采用）

外部研究成功时使用实时结果；失败时，由本地快照和确定性规则生成相同的公开产品协议。页面始终使用一套组件，只改变数据状态、日期、完整度和结论强度。

优点：外部服务完全不可用时仍能分析；不需要把私有持仓写入共享快照；四页交互一致。缺点：需要增加本地结果适配层，并严格区分外部事实与本地推断。

### 方案 B：只保存外部结果缓存

可以减少重复调用，但首次访问、缓存缺失或新股票仍会空白，不采用。

### 方案 C：只合并请求并延长缓存

可以降低额度消耗，但不能处理完全不可用场景，不采用。批量和缓存优化可在本方案稳定后单独实施。

## 交付架构

新增供应商中性的本地兜底适配层：

```text
浏览器请求
  -> 外部研究服务
  -> 成功：实时产品结果
  -> 失败：本地证据适配器
       -> TDX 快照 / 持仓账本 / 日报决策 / 公告与新闻
       -> 确定性研究结论
       -> 同一产品结果协议
  -> 页面渲染
```

本地适配层只负责把已有领域模型转换为研究工作台协议，不在 Web 模板中计算结论。优先复用现有 `workflows.py`、`analysis.py`、`deep_analysis.py` 和持仓分析模型，不复制第二套指标实现。

### 内部输入边界

- 大盘和机会继续优先读取每日共享研究快照。
- 个股兜底只读取用户选中的股票代码以及服务器本地股票快照。
- 持仓兜底由服务端根据当前登录账号的持仓路径加载；浏览器仍只提交代码和名称。
- 私有持仓的股数、成本、权重、备注和用户身份不得进入共享研究快照、前端 `data-*` 属性或外部研究请求。

## 产品协议

保留现有字段：

```text
ok / status / module / generated_at / verdict / action / primary_risk
findings / details / missing_sections / module_items / module_sections
decision_label / subject_count / coverage_ready / coverage_total
```

扩展交付状态：

```text
delivery: live / snapshot / stale_snapshot / local_fallback / unavailable
data_label: 实时研究 / 当日快照 / 本地证据 / 历史参考 / 数据缺失
fallback_reason: 对用户可读且不包含供应商、网关、trace 或凭证的降级原因
as_of: 实际证据时间
```

规则：

1. 只要本地存在一条可验证证据，`status` 至少为 `partial`，不得返回空白 `unavailable`。
2. 本地兜底不得伪造机构预期、研报观点、资金流或公告；缺失维度保留为待补。
3. 历史参考必须显示日期和过期状态，不能伪装成当日结论。
4. `fallback_reason` 只能说明“实时研究暂不可用，已使用本地证据”，不能展示外部服务品牌或原始错误。

交付优先级固定为：

- 大盘和机会：新鲜共享快照 -> 实时研究 -> 历史共享快照 -> 本地证据。
- 持仓和个股：实时研究 -> 本地证据。

## 四模块内容

### 每日大盘

保持当前主题优先结构：市场状态、前三主题、上涨/下跌/平盘/涨跌停分布、热门股票、指数趋势和退潮条件。

若外部研究失败，优先使用当日大盘与机会快照；两者都不可用时使用 TDX 市场快照生成市场宽度、板块方向和代表股。没有当日快照时显示历史参考日期并降低结论强度。

### 我的持仓

本地兜底必须展示：

- 组合结论：先处理风险、主题集中、主题分化、主题分散或数据待补。
- 主题暴露：最多八个主题，使用持仓数量和可用权重；公开协议不携带私有权重明细。
- 主题内分化：同一主题至少两只股票时比较近期趋势、盈亏、事件和数据完整度。
- 优先处理：最多三只，写明风险原因、复核条件和失效条件。
- 全部持仓：每只都有名称、代码、主题、状态、摘要和风险。

如果价格、成本或主题缺失，只降级相应判断，不隐藏整只持仓。

### 个股分析

本地兜底必须展示：

- 研究状态：继续跟踪、等待确认、基本面承压、证据不足。
- 核心结论：现状、原因和下一步验证写在一句话中。
- 三条依据：至少包含一条支持、一条反证或限制，以及一条下一步验证。
- 八维覆盖：财务质量、经营结构、机构预期、事件风险、行情资金、行业位置、公告事项、研报观点。
- 确认条件和失效条件：只能来自已有趋势、财务、事件或持仓边界，不生成无依据的目标价。

外部专属维度缺失时明确写“实时研究恢复后补充”，本地可用的财务、估值、行情、新闻和公告仍正常展示。

### 热点机会

保持“主线 -> 候选 -> 验证 -> 淘汰”的结构。外部研究失败时使用当日机会快照；快照不可用时从候选池、板块强度、成交、盈利变化和事件风险生成候选。

候选必须包含主题、入选依据、确认条件和失效条件；负面事件只能降级或排除候选，不能作为推荐理由。

## UI 设计

四页使用相同的状态带，不新增背景旁白：

```text
┌ 数据状态：本地证据 · 2026-07-15 · 部分维度待补 ┐
├ 结论标签 + 一句话判断                         ┤
├ 现在怎么做                    最大风险        ┤
├ 主题 / 分化 / 八维 / 候选内容                 ┤
├ 三条依据                                      ┤
└ 完整依据                                      ┘
```

状态颜色：

- `实时研究`：克制绿色。
- `当日快照`：墨蓝色。
- `本地证据`：琥珀色。
- `历史参考`：暖灰色。
- `数据缺失`：克制红色。

页面规则：

- 不再出现只包含“暂不可用”的空工作台。
- 降级提示只占一行，不覆盖核心结论。
- `module_sections` 与 `module_items` 有内容时必须展示；缺失维度使用待补卡，不隐藏整个区域。
- 重新分析按钮仍可尝试恢复实时研究；失败后保留当前本地内容，不把页面清空。
- 桌面首屏可看到结论、行动、风险和至少一个内容分区。
- 390px 下根节点不得横向溢出，状态带可换行，移动行动坞保持四个 44px 按钮。

## 失败与降级

| 场景 | 页面行为 |
| --- | --- |
| 外部额度耗尽 / 401 | 使用本地证据，状态为 `local_fallback` |
| 外部超时 / 5xx | 使用本地证据，保留可重试入口 |
| 当日共享快照存在 | 大盘和机会直接使用快照 |
| 本地股票快照部分缺失 | 展示已有维度，缺失维度标为待补 |
| 本地快照过期 | 使用 `stale_snapshot` 并显示实际日期 |
| 外部与本地均无数据 | 展示研究对象、缺口和恢复动作，不生成投资结论 |

## 安全边界

- 公网继续要求登录，匿名调用返回 401。
- 外部请求继续只携带允许的研究对象字段。
- 私有持仓不得写入共享快照或公共缓存。
- 页面和 JSON 不出现供应商品牌、能力 id、trace、请求头、API key 或网关地址。
- 动态页面继续使用安全 DOM API 和 `textContent`，不使用外部响应拼接 `innerHTML`。
- 不承诺收益，不自动交易，不输出无条件买入或卖出指令。

## 测试策略

按 TDD 实施，先增加失败测试：

1. 外部研究返回 `unavailable` 时，个股和持仓调用本地兜底。
2. 本地有部分证据时返回 `partial`、非空结论、分区和模块清单。
3. 重新分析失败后保留已有内容。
4. 个股缺少机构预期和研报时只降级相应维度。
5. 持仓兜底不把成本、股数、权重或账号写入公开 JSON。
6. 页面展示 `data_label`、日期和降级原因，但不出现底层供应商。
7. 大盘和机会原有快照优先级不回归。
8. 1280px、390px、键盘切换、详情展开和控制台无错误。

完成后运行当前 native 专项、legacy 回滚测试、全仓基线、Ruff、真实额度耗尽场景、桌面/移动浏览器和公网登录态验证。

## 验收标准

1. 四个模块在外部额度耗尽时仍有非空结论、行动、风险和内容清单。
2. 持仓页显示全部持仓、主题暴露、主题内分化和最多三只优先处理。
3. 个股页显示至少本地可用的财务、估值、行情、新闻、公告和明确缺口。
4. 大盘和机会继续优先使用每日共享快照，不增加无效外部调用。
5. 重新分析失败不会清空已经显示的兜底内容。
6. 页面明确展示证据状态和日期，不暴露底层服务或真实凭证。
7. 桌面和移动端均无横向溢出，四模块交互、搜索、下钻和详情展开正常。
8. 部署后本地、GitHub `main` 与服务器提交一致，服务、定时器和公网健康检查通过。
