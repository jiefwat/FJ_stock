# 四大工作台三秒精华版设计

## 问题

当前四页虽然已经删除背景话术，但首屏仍同时展示状态条、流程轨道、指标、情景、风险、证据和问财控件。每块内容都在强调自己，导致用户无法在 3 秒内回答“结论是什么、今天怎么办、最大风险是什么”。

## 目标

四个核心工作台默认只展示一层决策信息；专业诊断、方法过程、完整证据和外部问财保留在一个关闭的展开区中。

每页首屏必须回答：

1. 现在是什么状态。
2. 今天做什么或不做什么。
3. 最大风险是什么。
4. 最多三个需要关注的对象或证据。

## 方案比较

- 只缩字号和间距：内容数量不变，阅读压力更高，不采用。
- 直接删除专业内容：页面简单但失去复核能力，不采用。
- 首屏极简 + 一层展开：默认只保留决策，完整证据按需查看，采用。

## 四页默认内容

### 每日大盘

- 市场状态、风险预算、数据日期压成一条状态行。
- 唯一大结论：市场判断。
- 唯一行动：今天允许做什么。
- 唯一风险：最可能让判断失效的条件。
- 五步轨道、三情景、六维判断、行情明细和问财统一放入“展开市场依据”。

### 我的持仓

- 唯一大结论：组合状态和当前动作。
- 默认只展示风险优先级最高的 3 只持仓，每只只留名称、动作、原因、失效条件。
- 组合指标、风险暴露、其余持仓、持仓边界和问财放入“展开持仓依据”。

### 个股分析

- 唯一大结论：投资判断和当前动作。
- 默认只展示 3 条最高权重证据，同时包含至少 1 条反证或风险。
- 默认展示一个最关键失效条件；价格数据过期时不展示伪精确触发价。
- 决策轨道、诊断矩阵、情景、完整证据、研究动作和问财放入“展开完整档案”。

### 热点机会

- 唯一大结论：当前是否允许继续找机会。
- 默认只展示前 3 只研究候选，每只只留支持证据、最大反证和下一步。
- 证据漏斗、完整风险登记、其余候选、来源账本和问财放入“展开筛选依据”。

## 问财交互

问财研究坞改为默认关闭的 `<details>`：

```text
问财核查 · 按需展开                         已连接
```

展开后才显示快捷问题、上下文选择、输入框和结果。连接状态保留，但不再占用首屏大卡片。

## 信息与视觉规则

- 每页默认可见的三级标题不超过 4 个。
- 默认可见候选、持仓或证据不超过 3 项。
- 同一页面只能有一个大字号结论，其他内容使用普通字重。
- 删除默认可见的编号轨道、方法名、置信度解释和重复数据状态。
- 展开区最多一层，不嵌套多个“展开更多”。
- 桌面和 390px 都保持“结论 -> 动作/风险 -> 3 项 -> 展开依据”的顺序。

## 安全与行为边界

- 只改变默认展示，不改变研究计算、stale 闸门、持仓隔离、机会排序或问财技能路由。
- 问财仍只在用户主动展开并提交后调用，不做自动查询。
- 完整证据仍在 HTML 中可访问，避免为了简洁删除审计能力。
- 数据暂停时首屏只显示暂停动作和恢复条件，不显示历史精确价格动作。

## 验收标准

- 四页首屏均能在 3 秒内读到结论、动作和最大风险。
- 大盘不再默认显示五步轨道、三情景和六维卡片。
- 持仓默认最多 3 只；机会默认最多 3 只；个股默认最多 3 条核心证据。
- 四页问财控件默认关闭，点击一次即可展开。
- 1280px 和 390px 无横向溢出，移动端首屏不出现大段方法说明。
- 原有研究、鉴权、隐私、问财和数据闸门测试保持通过。
