# Aster Market 四模块分析甲板设计

## 决策

Aster Market 在现有公开只读市场地形之上增加四个互相独立、共享同一快照的数据模块：大盘分析、市场机会、股票分析和我的持仓。页面继续保持桌面最小宽度 `1180px`，不增加侧栏、手机代码、登录、交易、通知、AI 对话或后台写接口。

用户已明确授予产品与技术决策权，并声明后续方案默认同意，因此本设计直接视为批准。

## 方案比较

### 方案 A：四个独立页面

服务端分别渲染四个 URL。实现直接，但市场结论、机会和股票之间需要反复跳页，无法保持当前观察上下文。

### 方案 B：单一分析甲板（采用）

顶部命令带切换四个完整视图。所有模块共享交易日、数据状态和股票搜索；服务端继续输出 HTML 和只读 JSON，浏览器只负责切换、查询与本地持仓计算。它保持 Aster 的单工作面特征，也不引入前端构建链。

### 方案 C：重型前端 SPA

使用框架管理路由与状态。交互空间最大，但会增加依赖、构建步骤和客户端复杂度，不符合当前标准库边界。

## 信息架构

```text
┌──────────────────────────────────────────────────────────────────────┐
│ ASTER / 数据状态   大盘分析  市场机会  股票分析  我的持仓   搜索/刷新 │
├──────────────────────────────────────────────────────────────────────┤
│ 大盘分析：环境结论 / 市场地平线 / 四维证据 / 风险与观察点           │
│ 市场机会：机会走廊 / 阶段 / 证据 / 候选 / 失效条件                  │
│ 股票分析：搜索结果 / 趋势轨迹 / 估值资金 / 证据与风险               │
│ 我的持仓：本机持仓 / 市值盈亏 / 个股状态 / 组合暴露                 │
└──────────────────────────────────────────────────────────────────────┘
```

四个模块不是旧式编号工作区，也不是圆角卡片仪表盘。默认打开大盘分析；切换模块只改变可见甲板，不重新加载页面。URL hash 记录当前模块，刷新后可以恢复位置。

## 大盘分析模块

### 用户问题

1. 市场环境是扩张、轮动还是收缩？
2. 指数方向与市场广度是否一致？
3. 风险来自跌停压力还是主题集中？
4. 下一步要观察什么才会改变结论？

### 输出

- 环境结论：沿用确定性的扩张、轮动、收缩规则。
- 四维证据：指数方向、上涨参与度、涨跌停压力、主题集中度。
- 风险压力：低、中、高三级，只由快照数据计算。
- 观察点：根据当前状态给出验证条件，不给出买卖指令。
- 数据质量：交易日、生成时间、来源和缺失字段。

## 市场机会模块

### 机会走廊

主题按现有强度分数排序，最多展示八个。每个主题增加阶段标签：

- `扩散`：上涨占比不低于 `0.70`，连续天数不低于 2，且无高分歧。
- `加速`：涨幅不低于 `5%` 或成交变化不低于 `20%`，且无高分歧。
- `分歧`：`high_divergence=true`。
- `观察`：其他情况。

每条机会包含强度、参与度、成交变化、连续性、候选股票、证据摘要和失效条件。失效条件必须是可验证的数据描述，例如“上涨占比跌破 50%”或“分歧升高”，不得写成确定性价格预测。

## 股票分析模块

### 股票目录

从 `stocks` 与 `candidate_universe.items` 合并股票档案，按代码去重。只纳入至少有一根有效收盘价的股票。搜索支持代码、名称和主题，返回最多 20 个结果。

### 分析维度

- 趋势：最新价相对 5 日和 20 日均值；样本不足时明确标记。
- 动量：5 日和 20 日收益率，缺失时不补零。
- 波动：最近 10 根 K 线平均日内振幅。
- 估值：PE、PB、PS 和总市值，只展示快照已有值。
- 资金：成交额、换手率、内外盘差；不把成交额称为主力资金。
- 事件：最多五条股票新闻或公告摘要。
- 数据质量：可靠性、主来源和缺失字段。

输出只使用“强、平、弱”“完整、部分、有限”等观察标签，不生成目标价、买入、卖出或仓位建议。

## 我的持仓模块

### 隐私边界

公网应用没有登录，因此持仓代码、成本和数量只保存在当前浏览器的 `localStorage`，键名为 `aster.portfolio.v1`。数据不通过请求体、查询参数、日志或 Cookie 发送到服务器。

### 功能

- 添加股票代码、持仓数量和成本价。
- 修改或删除本机持仓。
- 调用公开股票分析接口获取名称、最新价、趋势和风险。
- 在浏览器计算市值、成本、浮动盈亏和收益率。
- 汇总主题暴露和单一持仓集中度。
- 空状态明确提示“持仓只保存在当前浏览器”。

服务器不提供持仓写接口，也不读取旧持仓文件。股票代码不存在或行情缺失时保留该行并显示原因，不虚构价格。

## 领域模型与数据流

新增不可变模型：

- `PriceBar`
- `StockProfile`
- `ValuationSnapshot`
- `FlowSnapshot`

`snapshot.py` 负责把原始 `stocks` 与候选档案映射成领域模型；`analysis.py` 只接收领域模型，输出市场分析、机会和单股分析字典；`presenter.py` 组合首页所需摘要。UI 不读取供应商字段。

```text
market_snapshot.json
  -> snapshot.py
  -> MarketSnapshot + StockProfile[]
  -> analysis.py
  -> market / opportunities / stock detail
  -> web.py JSON routes + ui.py shell
  -> app.js module switching + local-only portfolio
```

## HTTP 接口

- `GET /api/analysis/market`
- `GET /api/opportunities`
- `GET /api/stocks?query=<text>`
- `GET /api/stocks/<code>`

保留 `/`、`/healthz`、`/api/snapshot` 和静态资源接口。所有新增动态接口使用 `Cache-Control: no-store` 与现有安全头。未知股票返回 404 JSON；快照不可用返回 503；查询文本最多 40 个字符。

## 视觉系统

继续使用陶瓷白、墨石、钴蓝、信号橙、海藻绿和雾灰。字体继续使用压缩无衬线标题、Avenir 正文和等宽数据。

新的标志性元素是“分析甲板”：同一条市场地平线在大盘模块表示全市场，在个股模块收束为股票趋势轨迹；视觉语法一致但含义明确。市场机会使用连续横向走廊，不使用卡片网格。持仓模块使用账本式行布局和一条组合暴露带，不模拟券商交易终端。

唯一页面动画仍控制在一次 `420ms` 轨迹显现；切换模块使用短淡入，遵守 `prefers-reduced-motion`。页面不包含 viewport meta、手机导航、抽屉或 `@media (max-width: ...)`。

## 错误与降级

- 快照不存在或损坏：四模块统一展示数据中断，不生成示例价格。
- 股票没有足够 K 线：展示可用维度并标记样本不足。
- 估值或资金字段缺失：显示 `—`，不转成 0。
- 本地持仓 JSON 损坏：忽略损坏内容并显示恢复提示，不向服务器上报。
- 股票搜索无结果：给出可操作空状态。
- API 返回非 200：股票分析和持仓保留当前内容，并显示具体失败原因。

## 测试与验收

- 模型不可变与缺失值测试。
- 股票合并、K 线、估值、资金和数据质量解析测试。
- 大盘分析、机会阶段、股票趋势/动量/波动测试。
- 搜索、404、503、缓存头和供应商字段隔离测试。
- HTML 外部字符串转义和四模块结构测试。
- JavaScript 模块切换、股票搜索和 localStorage 持仓计算浏览器测试。
- 1280、1536、1920 三档桌面宽度无横向溢出。
- 公网四模块可访问，旧接口继续通过，主服务、Signal Desk 和 Nginx 保持健康。
