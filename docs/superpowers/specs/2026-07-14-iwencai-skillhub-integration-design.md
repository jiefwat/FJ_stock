# 问财 SkillHub 研究集成设计

## 目标

在不改变 StockTs 本地确定性研究结论的前提下，为个股页增加一个轻量研究追问入口。用户可以直接询问财务、机构预期、行业、事件、经营、股东、行情、公告、研报和新闻问题，服务端调用问财官方技能，并把结果压缩为可审计证据。

## 已核实的外部契约

- SkillHub 能力广场公开列出 160 个技能。
- “股票研究”版本为 `1.0.0`，UUID 为 `c72bf7b5-580d-487f-8977-2e82c1705671`。它是第三方编排说明，依赖 IBES、公司财务、历史价格和宏观 MCP 工具，不是独立 HTTP 对话接口。
- 问财官方数据技能通过 `POST https://openapi.iwencai.com/v1/query2data` 调用，使用 `IWENCAI_API_KEY` 和 `X-Claw-*` 请求头。
- 首期只接官方、无第三方配置的数据技能。社区技能不能直接改变 StockTs 评分、仓位或买卖结论。

## 方案比较

### A. StockTs 服务端技能路由器（采用）

服务端按问题路由到官方技能，返回结构化证据，再与当前个股档案并列展示。优点是契约真实、密钥不出浏览器、失败可降级、来源可审计；代价是首期不提供开放式多轮 Agent 推理。

### B. 在服务器运行 OpenClaw 代理

由 OpenClaw 加载“股票研究”并调用多个 MCP。编排能力强，但需要新增长期运行服务、模型、MCP 凭证和会话隔离，扩大攻击面，也不符合首期的简洁目标。

### C. 仅跳转问财网页

开发成本最低，但股票上下文割裂、无法保留本地 stale 闸门，也不能形成 StockTs 内部证据账本。

## 产品交互

个股档案的“关键证据”之后增加“问财研究追问”调度台：

```text
+-------------------------------------------------------------+
| 问财研究追问                已连接 / 未配置                  |
| [ 财务质量 ] [ 机构预期 ] [ 事件风险 ] [ 行业位置 ]         |
| [ 问：未来两年盈利预期是否改善？________________ ] [查询]   |
+-------------------------------------------------------------+
| 结论摘要                                                    |
| 问财事实      与本地档案的关系      未知与限制               |
| 来源：同花顺问财 · 使用技能 · 查询时间 · trace id 后 8 位   |
+-------------------------------------------------------------+
```

- 不展示欢迎语、背景故事、头像和空聊天记录。
- 默认建议问题与当前股票绑定，用户问题无需重复输入代码。
- 单次只调用一个最匹配技能；综合研究入口优先调用财务或机构预期，不在一次点击中并发消耗多个外部额度。
- 回答采用证据卡而非聊天气泡；最多显示 5 条数据、每条最多 6 个字段。
- 移动端为单列：状态、快捷问题、输入、结论、事实、来源。
- 键盘可提交，加载期间按钮禁用，支持 `prefers-reduced-motion`。

## 技能路由

| 意图 | 官方技能 | 页面入口 |
|---|---|---|
| 财务、盈利、现金流、估值 | `hithink-finance-query` | 个股 |
| 一致预期、评级、业绩预测 | `hithink-insresearch-query` | 个股 |
| 行业估值、行业排名 | `hithink-industry-query` | 个股、大盘 |
| 业绩预告、质押、解禁、监管 | `hithink-event-query` | 个股 |
| 主营、客户、供应商、合同 | `hithink-business-query` | 个股 |
| 股东、股本、实控人 | `hithink-management-query` | 个股 |
| 行情、资金流、技术指标 | `hithink-market-query` | 个股 |
| 公告、研报、新闻 | `announcement-search`、`report-search`、`news-search` | 个股 |
| 指数、宏观 | `hithink-zhishu-query`、`hithink-macro-query` | 大盘 |
| 板块筛选 | `hithink-sector-selector` | 热点机会 |
| A 股筛选 | `hithink-astock-selector` | 热点机会 |

首期实现个股入口和上述个股技能；大盘、热点机会只保留后续可复用路由，不增加页面复杂度。

## 服务端结构

- `src/stock_ts/iwencai.py`：技能目录、意图路由、OpenAPI 客户端、响应压缩与安全配置摘要。
- `POST /api/iwencai/research`：接收 `code`、`name`、`question` 和本地数据日期，验证登录、长度、Content-Type 与请求体大小。
- `src/stock_ts/webapp/stock_workspace.py`：只渲染研究调度台 HTML，不读取密钥、不做研究判断。
- `src/stock_ts/webapp/shell.py`：提交请求、渲染状态和错误，不持久化问题或回答。

## 数据流

```text
问题 + 当前股票
  -> 同源 POST endpoint
  -> 意图路由
  -> 服务端生成 64 位 trace id
  -> 问财官方技能 OpenAPI
  -> 压缩动态字段
  -> 事实 / 关系 / 未知 / 来源 JSON
  -> 页面证据卡
```

本地档案日期只用于提示数据时效和冲突边界，不发送持仓数量、成本、用户名、Cookie 或其他私人信息。

## 安全与失败边界

- Key 仅从 `IWENCAI_API_KEY` 读取，配置摘要只能返回 `configured` 或 `missing`。
- Base URL 默认固定为 `https://openapi.iwencai.com`；只允许 `https`，不接受浏览器传入 URL。
- 问题去除控制字符，长度限制 200 字；请求体限制 16 KiB；外部超时 12 秒。
- 外部返回最多读取 1 MiB，最多展示 5 条数据；不把原始 HTML 注入页面。
- 网关错误、无 Key、超时或无数据均返回可读降级，个股主页面保持可用。
- 问财结果是外部证据，不得绕过 stale 闸门，不自动改写仓位，不提供自动交易。
- 公网只允许已登录用户调用，并使用进程内按客户端地址的短周期限流。

## 验收标准

- 无 Key 时页面显示“未配置”，提交后返回配置提示且不泄露环境变量。
- 有 Key 时请求头、请求体、trace id 和技能版本符合官方契约。
- 财务、预期、行业、事件、经营、股东、行情、公告、研报、新闻问题能路由到正确技能。
- 任意外部数据与错误文本均经过 HTML 安全输出；响应不包含 Key。
- 桌面和 390px 页面信息顺序清晰，不增加背景旁白。
- 专项测试、Web 回归、lint 通过；全量测试只允许记录已知基线失败。
- 本地 `main`、`origin/main`、服务器 `main` 一致，公网健康检查和真实页面可访问。
