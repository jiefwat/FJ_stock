# 个股研究决策链与页面 UI V2 设计

## 1. 目标

把个股页从“指标和风险罗列”升级为一条可审计、可执行、可证伪的研究决策链，让用户在 30 秒内回答：

1. 这只股票当前最关键的投资矛盾是什么？
2. 哪些事实支持或反对当前判断，权重如何？
3. 现在做什么，什么条件下转强、降级或退出？

系统不承诺收益，不生成无依据目标价，不把技术突破写成公司质量改善。

## 2. 当前问题

- 主结论主要拼接财务、技术和事件摘要，缺少从事实到结论的因果链。
- 最强证据默认偏向技术结构，研究权重没有区分“决定投资逻辑”和“只决定执行时点”。
- 预期差与估值虽有字段，但没有进入专业档案的主决策链。
- 行业上下文只影响完整度分，未进入结论；市场上下文在构建档案时被丢弃。
- 入场、加仓、减仓和失效主要是价格条件，没有绑定盈利、事件和估值假设。
- 页面同时展示决策轨道、风险、执行、诊断、情景和证据，信息顺序仍接近长报告。

## 3. 方案选择

### 方案 A：只优化文案和卡片样式

改动最小，但不能解决研究浅、结论缺少因果和权重的问题，不采用。

### 方案 B：新增更多分析模块

可以增加行业、估值和预测卡片，但会重新膨胀页面，且各模块仍可能互相平行，不采用。

### 方案 C：重构为“研究假设驱动的决策链”

保留现有数据源和安全闸门，新增结构化研究假设与证据权重，再围绕同一模型重排页面。该方案同时解决方法和展示问题，采用。

## 4. 研究方法

### 4.1 一句话研究假设

每只股票必须形成一句可证伪假设，而不是评分结论。假设由以下信息组合：

- 当前经营状态；
- 最大事件或治理风险；
- 估值是否有可比锚；
- 价格结构是否只是在反弹，还是已完成验证。

缺少经营、估值或事件证据时，假设必须显式降级为“待验证”，不能用技术强度补齐。

### 4.2 核心矛盾与因果链

研究链固定为三段：

```text
经营/事件事实 -> 盈利或风险如何变化 -> 估值与价格是否确认
```

技术和资金默认只负责“确认执行时点”，不能单独证明公司质量。高等级事件风险可以直接否决新开仓。

### 4.3 证据权重

证据分为五类：

| 证据 | 默认重要性 | 作用 |
| --- | --- | --- |
| 盈利质量 | 高 | 决定研究假设是否成立 |
| 估值与预期差 | 高 | 判断价格是否已反映经营变化 |
| 事件与治理 | 高 | 识别催化和一票否决风险 |
| 行业位置 | 中 | 判断变化是否有相对优势 |
| 资金与价格 | 中 | 确认交易时点和失效线 |

每类证据必须同时输出：`事实`、`方向`、`推断`、`未知项`。方向只允许 `支持`、`中性`、`反证`、`未知`，避免伪精确分数。

### 4.4 预期差和估值匹配

- 未接入一致预期时，明确写“预期差不可量化”，不把新闻标题或涨跌幅伪装成一致预期。
- 亏损时 PE 失去解释力，估值判断转向 PB、PS、资产质量和盈利修复条件。
- 估值结论必须与增长、现金流和盈利质量同屏表达；低估值不能单独构成买点。
- 有行业中位数或历史分位时可以展示相对位置，但仍需说明对应的经营条件。

### 4.5 情景与价格

改善、基准、恶化三种情景都必须包含：

- 经营或事件前提；
- 价格确认位置；
- 对应动作；
- 研究失效条件。

禁止只用“涨、震荡、跌”描述情景。

### 4.6 持仓身份差异

- 未持仓：先过研究确认，再过价格确认，最后才允许小仓验证。
- 已持仓：优先按成本、仓位和失效条件管理，不因亏损自动补仓。
- 数据过期：仓位上限与新增风险预算均为 0，价格线全部暂停使用。

## 5. 结构化模型

新增两个只读模型：

### `ThesisFramework`

- `headline`：一句话研究假设；
- `core_conflict`：当前最关键矛盾；
- `causal_chain`：三段因果链；
- `expectation_gap`：预期差是否可量化；
- `valuation_fit`：估值与盈利质量是否匹配；
- `catalyst_window`：下一次有效验证窗口；
- `key_unknown`：最需要补齐的未知项；
- `falsifier`：什么事实会推翻假设。

### `WeightedEvidence`

- `dimension`：证据类别；
- `importance`：高或中；
- `direction`：支持、中性、反证、未知；
- `fact`：可回溯事实；
- `inference`：该事实如何影响假设；
- `unknown`：尚未得到的关键证据。

`ProfessionalStockDossier` 组合这两个模型，Web 只渲染，不在页面层推导结论。

## 6. 页面信息架构

首屏只保留四块：

```text
[股票身份 / 现价 / 数据日期 / 数据闸门]

[投资判断]
立场 + 当前动作 | 一句话假设 + 核心矛盾 + 完整度 + 周期

[决策条件]
当前判断 | 研究转强 | 价格确认 | 降级条件 | 论点失效

[执行边界]
当前动作 | 仓位/风险预算 | 入场触发 | 减仓/退出 | 禁止动作
```

首屏之后按以下顺序展示：

1. 论点链：事实 -> 推断 -> 验证；
2. 关键证据：五类加权证据；
3. 风险反证：最多三项真正可能推翻结论的风险；
4. 三情景：改善、基准、恶化；
5. 证据账本：完整诊断、来源、日期、限制和原始分析，默认关闭。

“诊断底稿”和多角色方法说明不再占据主页面。方法说明属于背景旁白，原始事实保留在证据账本。

## 7. 视觉方向

延续当前蓝灰研究终端，不引入新的全站视觉体系。

- `深海蓝 #10283D`：主判断；
- `研究金 #B4853A`：关键条件；
- `证据绿 #2C8061`：支持；
- `风险红 #B4483D`：反证与失效；
- `雾蓝 #EDF2F5`：页面底色；
- `墨蓝 #13273A`：正文。

页面的唯一视觉签名是“论点脊柱”：三段因果链沿一条有方向的轨道排列，结构本身表达研究逻辑，不使用装饰性编号。桌面横向，手机纵向。

## 8. 降级与安全

- stale 或 blocked 行情维持硬闸门，置信度为 0，禁止新增风险。
- 缺少一致预期时明确标注不可量化。
- 缺少财务、估值、行业或事件数据时，将对应证据标为未知。
- 事件标题只作为风险扫描，仍提示打开原文复核。
- 所有动作均为条件化研究边界，保留“不构成投资建议”。

## 9. 验收标准

- 个股档案包含一句话假设、核心矛盾、三段因果链和五类加权证据。
- 每类证据明确区分事实、推断和未知项。
- 亏损股票不会使用 PE 低估逻辑，且价格突破不能覆盖盈利或高风险事件反证。
- 入场和加仓条件同时包含研究确认与价格确认。
- 已持仓和未持仓给出不同动作边界。
- stale 数据下不出现可执行旧价格触发线。
- 页面首屏只有身份、投资判断、决策条件和执行边界四类信息。
- 桌面和 390px 手机宽度无横向溢出，证据账本默认关闭。
- 专项测试、Web 测试、lint 通过；全量测试中的既有失败单独记录。

## 10. 不做事项

- 不接券商下单或自动交易。
- 不新增收益承诺、上涨概率或无依据目标价。
- 不新增 LLM 自动编写研究结论。
- 不修改持仓数据、行情快照、日报产物、Signal Desk、DSA、Nginx 或服务器定时任务。
