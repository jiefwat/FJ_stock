# 四模块分析甲板代码审查

## 审查范围

审查 `codex/aster-analysis-modules` 相对已部署 Aster 基线的全部模型、解析、分析、API、HTML、CSS、JavaScript、测试和文档改动。

## Findings

未发现仍未修复的 P0、P1 或 P2 问题。

审查中发现并修复：

1. JavaScript 的 `Number(null)` 会把缺失估值或零成本收益误显示为 `0.00`。现已增加空值保护，统一显示 `—`，并补充资源契约测试。
2. 候选池拥有而股票主档案缺失的估值、资金或质量字段没有合并。现已增加候选补全规则和回归测试。
3. 股票新闻已解析但公告未进入事件证据。现已合并新闻与公告，并补充解析测试。
4. 股票详情和持仓空状态的 CSS 会覆盖原生 `hidden`。现已增加显式状态规则和回归测试，并通过真实浏览器复验。

## Assumptions

- 运行时快照继续提供当前已验证的市场、主题、候选和股票字段。
- 公网应用保持无登录，因此持仓不会迁移到服务端。
- 现有代理继续转发本机 8501，本次不修改代理配置。

## Residual Risks

- 浏览器持仓依赖 localStorage，清理站点数据或更换浏览器会丢失，设计上不做跨设备同步。
- 生产股票档案的数据质量不一致，模块会降级显示但不能补齐上游缺失数据。
- 当前标准库服务适用于只读单机流量；未来若增加用户系统或写操作必须重新设计。

## Decision

实现满足首次四模块公网发布门禁，可进入最终验证和部署。
