# Aster Market 桌面舒适度代码审查

## 审查范围

审查 `origin/main` 到当前工作区的桌面布局、浏览器交互、持仓计算、快照缓存、HTTP 接线、测试与运维文档。

## Findings

未发现未修复的 P0、P1 或 P2 问题。

审查过程中已修复以下问题：

- 全局平滑滚动与模块位置恢复互相竞争，连续切换会记录动画中间位置。
- 静态资源版本不统一，刷新后可能继续执行旧交互脚本。
- 旧个股请求晚失败时可能覆盖新股票详情的状态。
- 持仓异步渲染缺少序号保护，编辑或删除时可能把旧行情配到新账本。
- 持仓行情缺失曾按价格 0 参与汇总，可能制造虚假的组合亏损。

## Open Questions / Assumptions

- 生产行情继续使用原子替换；缓存指纹合同不支持刻意保留 inode、大小和修改时间的原地改写。
- 产品继续保持公开只读、无账户、无交易接口，持仓仅存在当前浏览器。
- 页面继续只支持最小宽度 1180px 的桌面浏览器。

## Residual Risks / Testing Gaps

- 原生 JavaScript 没有独立单元测试运行器；关键竞态由源码契约、语法检查和真实浏览器正常链路共同覆盖。
- 公网真实 84MB 快照的冷启动与热缓存性能需要在部署后记录。

## Decision

通过本地提交门禁；公网性能和服务健康验证完成后可结束发布。
