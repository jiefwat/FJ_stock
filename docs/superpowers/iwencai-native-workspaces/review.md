# 安全与产品审查

## 已确认

- 默认根页面在创建行情 Provider 前返回新研究壳，旧四模块分析不会在首屏运行。
- 浏览器不能指定网关、能力 id 或版本；服务端只使用固定 allowlist。
- 产品 JSON 不包含供应商名称、能力 id、trace、网关、授权头或密钥。
- 动态数据行会丢弃 `trace / skill / provider / authorization / api_key / gateway / channel` 等内部字段。
- 持仓研究只传代码和名称，最多 3 只；股数、成本、权重、行业备注、账号和 Cookie 不进入研究请求。
- 登录、用户/IP 限流、16 KiB 请求上限、12 秒单能力超时和 1 MiB 单响应上限保留。
- 结果只给条件化研究动作，不自动交易，不承诺收益；全部失败时不沿用旧结论。

## 已知边界

- 研究能力返回动态表格，产品层按能力字段语义、优先级和最低证据要求确定性提取；未知字段不进入首屏，不伪造估值或买卖结论。
- 组合诊断不接收真实仓位权重，因此只负责逐只风险优先级，不输出组合再平衡比例。
- 本地凭证只保存在 Git 忽略且权限收紧的 `.env`，真实调用已验证；凭证未进入提交、HTML、JSON、日志或测试样例。
- 官方 HTTPS 客户端显式使用 `certifi` 可信 CA，避免 macOS Framework Python 缺少系统 CA 时四模块同时失败；服务端仍只连接固定官方域名。
- 当前宏观查询会返回指数报价，语义门禁将其降级为缺口；当前持仓机构预期也可能只有身份和报价，同样不会冒充完整证据。
- 旧页面保留在 `STOCK_TS_WEB_VERSION=legacy` 后，作为紧急回滚，不是默认路径。
