# 实现审查

## 结论

本次改造解决的是证据交付链，不是用静态文案遮住错误。外部研究仍优先；额度、超时、网关或配置异常时，四个工作台使用本地行情、财务、资金、公告、持仓、候选和研究快照生成同一产品协议。

## 已确认

- `portfolio` 和 `stock` 不读写共享研究快照，私有持仓只通过当前登录账号的服务端路径进入本地分析。
- 浏览器只能传代码、名称、板块和刷新标记；股数、成本、权重、备注、账号和文件路径不进入公开协议。
- 持仓公开摘要不包含股数、成本、权重、备注或账本路径。
- 本地没有的机构预期、研报、公告或经营证据标记为待补，不生成目标价，不伪造数据。
- 大盘和机会继续保持共享快照优先，历史快照明确显示为“历史参考”。
- 候选下钻优先使用独立个股快照，其次本地候选 K 线，最后复用机会快照中的真实候选依据。
- 页面只显示供应商中性的状态、结论、动作、风险和依据。
- 刷新异常不会清空已经可读的结果。

## 风险与边界

- 本地结论是条件化研究结论，不等于交易信号，也不承诺收益。
- 如果本地个股只有机会快照，页面会明确标记“等待确认”，覆盖度为 `2/8`，不会把候选依据冒充完整公司研究。
- 历史快照的时点会直接展示，用户需要结合日期判断时效。
- 全仓默认模式仍保留 141 项既有红灯；发布判断依赖专项、Legacy 回滚、失败集合不增长、真实 API 和浏览器五层证据。

## 凭证检查

真实 API Key 只从本机和服务器环境读取，没有进入提交、测试数据、HTML、JSON 或日志。公开响应扫描未发现供应商品牌、能力 ID、trace、网关或授权字段。
