# 完整股票研究系统 AI Review

## 审查范围

`main...codex/complete-research-system` 的研究引擎、外部能力适配边界、快照、每日任务、Web API、前端渲染和测试。

## Findings

- [已修复 P1] 持仓缓存键原来只包含前三只股票，不同组合可能误复用研究结果；已与 20 只分析上限保持一致，并增加隔离回归测试。
- [已修复 P1] 新闻、公告和研报未执行 30 天首屏新鲜度门禁；现在旧证据只保留在完整依据，不参与判断、风险和前三条发现。
- [已修复 P1] 快照写入共用固定 `.tmp`，网页刷新与定时任务并发时可能抛出 `FileNotFoundError`；现在使用同目录唯一临时文件、刷新并原子替换。
- [已修复 P2] 实时服务直接抛异常或配置临时缺失时不能回退历史快照；现在全局模块返回明确的 `stale_snapshot`。
- [已修复 P2] 大盘区间跌幅被压缩成普通涨跌幅，和当日涨跌看似冲突；现在保留区间日期，并在指数卡同时展示当日和区间趋势。
- [已修复 P2] 个股八维卡片主标题全部显示股票名；现在显示财务质量、经营结构、机构预期、事件风险、行情资金、行业位置、公告事项和研报观点。

## Open Questions / Assumptions

- 公网继续要求登录，外部研究请求不开放匿名调用。
- 大盘和机会可使用共享快照；持仓和个股只做登录态实时研究，不写共享快照。
- 服务端时区已确认是 `Asia/Shanghai`，定时点按北京时间执行。

## Residual Risks / Testing Gaps

- 问财动态字段可能随上游变化；规范化层会把证据不足降级，但仍需观察线上命中率。
- 全量测试仍有主分支既有的 legacy UI 和旧流水线失败；本分支没有新增失败，专项契约已全部通过。
- 本系统是研究辅助，不自动交易；候选和动作必须结合失效条件复核。

## Decision

未发现未修复的 P0/P1/P2 阻断项，可以进入 `main` 和公网部署。
