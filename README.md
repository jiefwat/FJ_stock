# Market Desk

本地优先的 A 股投研工作台。核心路径是：判断市场状态 -> 读取市场事件 -> 发现机会 -> 核对个股证据 -> 管理跟踪与持仓。

## 模块

- 今日：市场温度、研究风险预算、市场异动、指数、板块与下一步动作。
- 大盘：涨跌广度、指数、评分因子、行业热度、市场事件雷达与板块核验。
- 机会：四套策略预设、策略诊断、候选决策卡、风险排除与 CSV 导出。
- 个股：代码/名称搜索、K 线技术结构、投资建议、横纵向比较、正反证据与失效条件。
- 持仓：本地保存持仓数量、成本、目标仓位、持仓逻辑、调仓建议和风险旗标。
- 跟踪：没买或还没决定买的股票，保存关注理由、跟踪状态和放弃条件。
- 数据：数据源、覆盖率、时效、自动刷新状态、降级规则与手动刷新。

## 启动

```bash
make setup
make start
```

默认打开 <http://127.0.0.1:8765>；如果端口已被其他程序占用，启动脚本会自动选择后续空闲端口并打印实际地址。停止服务：

```bash
make stop
```

开发模式使用 `make dev`，前端为 `http://127.0.0.1:5173`，API 为 `http://127.0.0.1:8000`。

## 验证

```bash
make test
make verify
```

`make verify` 会检查 Python lint/type/test、前端 type/test/build，并从真实数据源验证 A 股数量、核心字段覆盖率、指数集合、行业数量和数值有效性。公网服务默认每 2 小时自动刷新一次市场数据，也可在数据页手动刷新。

## 数据来源与限制

- A 股全市场快照：新浪公开行情列表。
- 核心指数与个股复权日线：腾讯公开行情接口。
- 行业热度：新浪行业行情与东方财富板块资金流增强。
- 市场事件：东方财富公开快讯，用于事件分类、影响说明和核验清单。
- 语义研究：可选增强项；未配置端点和 API Key 时不会影响核心功能。
- AInvest：仅用于产品交互参考，不是数据依赖。

公开接口可能限流或调整。服务优先展示最后一次有效快照并标记时效；没有快照时明确显示不可用。当前产品用于个人投研辅助，不构成投资建议。公开或商业部署前需要重新核查数据授权与服务条款。
